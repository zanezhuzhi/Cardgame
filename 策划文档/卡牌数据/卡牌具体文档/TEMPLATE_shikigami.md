# 式神卡文档模板

> **使用说明**：复制此模板创建新式神文档，文件名格式 `shikigami_XXX_名称.md`，与 `cards.json` 的 `id` 一致

---

## 📋 基础信息

| 属性 | 值 |
|:----:|:---|
| ID | shikigami_XXX |
| 名称 | 【式神名称】 |
| 类型 | shikigami (式神) |
| 稀有度 | SSR / SR / R |
| 声誉 | +N（cards.json 中 charm 字段） |
| 功能分类 | 【如：输出/连击、防御/爆发 等】 |
| 技能名 | 【技能名称】 |
| 多人标记 | 是/否（cards.json 中 multiPlayer 字段） |

### 技能类型说明

| 技能类型 | 标记 | 说明 |
|:--------:|:----:|------|
| 主动技能 | 【启】 | 需玩家主动发动，消耗鬼火 |
| 触发技能 | 【触】 | 满足特定条件时可触发 |
| 被动技能 | 【自】 | 满足条件时自动执行 |
| 永久被动 | 【永】 | 只要在场上就持续生效 |
| 妨害标签 | [妨害] | 对其他玩家产生负面影响 |

### 技能消耗

| 消耗类型 | 格式 | 示例 |
|----------|------|------|
| 固定鬼火 | 鬼火-N | 鬼火-2 |
| 可变鬼火 | 鬼火-N（N≥1） | 鬼火-N：伤害+N+1 |
| 弃置手牌 | 弃置N张手牌 | 弃置N张手牌：伤害+N |
| 超度手牌 | 超度N张手牌 | 超度1张手牌 |

```

输出: 是否发动技能、选择哪些目标/选项

决策逻辑:
1. 检查发动条件（鬼火是否足够）
```

### 涉及文件

- `shared/data/cards.json` - 式神基础数据
- `shared/game/effects/ShikigamiEffects.ts` - 技能效果实现
- `shared/game/effects/ShikigamiEffects.test.ts` - TDD测试用例
- `server/src/game/MultiplayerGame.ts` - 游戏流程管理（如需）

### 实现代码

```typescript
// 技能效果注册
registerShikigamiSkill('【技能名】', async (ctx) => {
  const { player, gameState, triggerInteract } = ctx;
  
  // 1. 检查发动条件
  if (player.ghostFire < N) {
    return { success: false, error: '鬼火不足' };
  }
  
  // 2. 支付消耗
  player.ghostFire -= N;
  
  // 3. 执行效果
  // ...
  
  return { success: true, message: '【效果描述】' };
});
```

### 注意事项

1. **技能发动校验**：必须在效果入口做前置校验
2. **日志格式**：
   - 发动: `⚡ ${player.name} 发动【${skillName}】`
   - 效果: `🔮 ${effectDescription}`
3. **状态同步**：多人游戏需通过 `notifyStateChange()` 广播
