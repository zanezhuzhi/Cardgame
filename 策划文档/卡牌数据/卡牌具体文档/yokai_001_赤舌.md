## 📜 效果原文

> 效果：每名对手从自己的弃牌堆中选择1张基础术式或招福达摩，置于自己牌库顶。

---

## 🔍 效果分解

1. **遍历每名对手** — 对除打出者以外的每位玩家执行
2. **筛选弃牌堆** — 在该对手的弃牌堆中，筛选出「基础术式」(spell_001) 或「招福达摩」(token_001) 的卡牌
3. **对手选择1张** — 由该对手从符合条件的卡牌中选择1张（多人模式由对方玩家选择；单人模式由AI自动选择）
4. **置于牌库顶** — 将选中的牌从弃牌堆移到该对手的牌库顶部
5. **无符合条件** — 若弃牌堆中没有符合条件的牌，跳过该对手

---

## 🎮 单人模式实现

### 逻辑描述

单人沙盒模式只有1名玩家，**没有对手**。赤舌的妨害效果在单人模式下**无目标可执行**。

处理方式：效果正常执行但循环体不进入，返回提示信息"单人模式无对手，妨害效果跳过"。

### 涉及模块

- `shared/game/effects/YokaiEffects.ts` — 御魂效果处理器

### 风险点

- 单人模式下效果等同于「无效果」，但卡牌本身仍贡献 +1 声誉
- 需确保不会因为无对手而报错或返回 `success: false`

## 🌐 多人模式实现

### 逻辑描述

```
1. 获取所有对手玩家列表
2. for each 对手:
   a. 从对手的 discard[] 中筛选 name === '基础术式' || name === '招福达摩' 的卡牌
   b. if 符合条件卡牌数 === 0 → 跳过
   c. if 符合条件卡牌数 === 1 → 自动选中
   d. if 符合条件卡牌数 > 1 → 等待对手玩家选择1张
   e. 将选中卡牌从 discard 移到 deck 顶部（push）
3. 效果结束
```

### 涉及模块

- `shared/game/effects/YokaiEffects.ts` — 效果逻辑
- `server/src/game/MultiplayerGame.ts` — 多人等待对手选择
- `client/src/App.vue` — 对手选择弹窗UI

### 交互机制

1. **服务端→被妨害玩家**：发送 `game:selectFromDiscard` 事件，携带符合条件的弃牌列表
2. **被妨害玩家客户端**：显示弃牌堆选择弹窗
3. **被妨害玩家→服务端**：发送选择结果
4. **服务端**：移动卡牌到该玩家牌库顶，处理下一个对手
5. 所有对手处理完毕后继续游戏

### 风险点

- 需要顺序处理多名对手的选择（不能并行，因为涉及状态依赖）
- 对手掉线时应自动选第一张符合条件的牌
- 其他玩家不应看到对手弃牌堆的具体内容

```typescript
  // 对手选择后置于牌库顶

```

### 当前实现问题

| # | 问题 | 描述 |
|---|------|------|
| 1 | 筛选条件错误 | 用 `c.tags?.includes('基础')` 筛选，但实际数据中基础术式没有 tags 字段，应改为 `c.name === '基础术式'` |
| 2 | 自动选第一张 | 规则要求"对手选择"，当前直接取 `validCards[0]`，应让对手选择（多人模式） |
| 3 | 单人无提示 | 单人模式下无对手时返回的消息不明确 |

### 修复方案

```typescript
registerEffect('赤舌', async (ctx) => {
  const { player, gameState } = ctx;
  const opponents = gameState.players.filter(p => p.id !== player.id);
  
  if (opponents.length === 0) {
    return { success: true, message: '[妨害] 赤舌：无对手，效果跳过' };
  }
  
  let affected = 0;
  for (const opp of opponents) {
    const validCards = opp.discard.filter(c => 
      c.name === '基础术式' || c.name === '招福达摩'
    );
    if (validCards.length === 0) continue;
    
    // 只有1张时自动选择，多张时由对手选择（单人模式自动选第一张）
    let selectedId: string;
    if (validCards.length === 1) {
      selectedId = validCards[0]!.instanceId;
    } else {
      selectedId = validCards[0]!.instanceId; // 单人模式自动选第一张
    }
    
    const idx = opp.discard.findIndex(c => c.instanceId === selectedId);
    if (idx !== -1) {
      opp.deck.push(opp.discard.splice(idx, 1)[0]!);
      affected++;
    }
  }
  
  return { 
    success: true, 
    message: `[妨害] 赤舌：${affected}名对手将弃牌堆卡牌置于牌库顶` 
  };
});
```

### 注意事项

1. 筛选条件必须用 `c.name` 匹配「基础术式」和「招福达摩」，不能用 tags
2. 「置于牌库顶」用 `deck.push()`（牌库顶 = 数组末尾，与 `drawCards` 的 `deck.pop()` 对应）
3. 单人模式下无对手时应返回 `success: true`（效果合法但无目标）
4. 声誉 +1 由卡牌数据本身提供，不在效果中处理

## ⏱️ 强制中止行为

| 中止时机 | 已执行的部分 | 默认处理 |
|----------|------------|----------|
| 效果触发前 | 无 | 效果跳过，卡牌正常进入弃牌堆 |
| 遍历对手过程中（某名对手等待选择时超时） | 之前的对手已处理完毕 | 当前对手自动选第一张符合条件的牌，后续对手继续正常处理 |

### 默认选择策略

赤舌效果中的选择由**对手**做出（选择自己弃牌堆中的牌），不是打出者选择：

| 选择类型 | 超时默认 |
|----------|----------|
| 对手选择弃牌堆中的1张 | 自动选择符合条件的第一张牌 |

### 事务性

- 已处理完的对手不回滚
- 所有对手处理完毕后效果结束，不存在中间状态

## 🎲 AI接管行为

### AI决策表

赤舌是妨害卡，决策点在**被妨害的对手**一侧：

| 决策点 | AI身份 | AI策略 |
|--------|--------|--------|
| 打出赤舌 | AI为打出者 | 无决策（效果自动对所有对手生效） |
| 对手选择弃牌堆中1张牌置于牌库顶 | AI为被妨害者 | 选择对自己影响最小的牌：优先选「招福达摩」（弱牌），其次选「基础术式」 |

### 策略说明

被妨害的 AI 对手选牌优先级（L1 规则策略）：
1. 优先选「招福达摩」— 声誉+1 但不能打出，放牌库顶影响较小
2. 其次选「基础术式」— HP=1 伤害=1，是最弱的阴阳术
3. 若两种都有多张，选第一张（无需进一步区分）
