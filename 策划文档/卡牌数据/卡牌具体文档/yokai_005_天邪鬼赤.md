## 📜 效果原文

> 效果：伤害+1，弃置任意数量手牌，然后抓等量的牌。

---

## 🔍 效果分解

1. **伤害+1** — 立即 `damage += 1`（始终执行）
2. **可选换牌** — 玩家可选择弃置 **0～当前手牌张数** 中的任意子集；每弃置 1 张，之后从牌库顶抓 1 张（等量置换）
3. **无手牌** — 跳过步骤 2，仅保留伤害+1

---

## 🎮 单人模式实现

### 逻辑描述

先加伤害；若手牌非空，弹出多选手牌界面（允许选 0 张 = 不换牌）；对每张被选中的牌：手牌 → 弃牌堆，再执行等量 `drawCards`。

### 涉及模块

- `shared/game/effects/YokaiEffects.ts` — `onSelectCards(hand, maxCount)`，`maxCount = hand.length`
- `client/src/App.vue` / 单人管线 — 多选确认与空选支持

### 风险点

- **弃置与抓牌顺序**：须先完成全部弃置（或按规则逐张弃-抓）；当前实现为一次性弃置 N 张再抓 N 张，与策划「弃置任意数量然后抓等量」一致
- 牌库不足时，少抓即可，不回收已弃置的牌

## 🌐 多人模式实现

### 逻辑描述

同单人；服务端更新打出者 `damage`、`hand`、`discard`、`deck` 并广播。

### 涉及模块

- `server/src/game/MultiplayerGame.ts` — 可打出与效果结算
- `策划文档/交互设计.md` §4.2：**多效果卡**，伤害+1 为保底，换牌需有手牌

### 风险点

- 等待选牌超时：默认「不弃任何张」或按规范「必须选则选第一张」— 本效果 **允许 N=0**，默认应为 **空数组**（不换牌）

## 💻 代码设计

### 涉及文件

- `shared/game/effects/YokaiEffects.ts` — `registerEffect('天邪鬼赤', …)`

### 关键逻辑要点

```text
damage += 1
if (hand.length > 0 && onSelectCards) {
  ids = await onSelectCards(hand, hand.length)  // 允许 length 0～N

  drawCards(player, ids.length)

```

### 注意事项

- 打出后本卡已离开手牌；「任意数量」指 **打出后剩余手牌**
- 与交互设计：**伤害+1 ✅ | 换牌（需手牌）**

## ⏱️ 强制中止行为

| 中止时机 | 已执行的部分 | 默认处理 |
|----------|------------|----------|
| 伤害+1 之后、等待选牌时中止 | 伤害已 +1 | **不弃牌**（`onSelectCards` 视为空数组），不额外抓牌 |

### 默认选择策略

| 选择类型 | 默认行为 |
|----------|----------|
| `onSelectCards`（多选手牌弃置） | **空数组**（0 张）；本效果允许 0，符合损失最小 |

## 🎲 AI接管行为

### AI决策表（L1）

| 决策点 | AI策略 |
|--------|--------|
| 弃哪些手牌 | 《卡牌开发》**弃置价值最低的牌**：HP 升序，同 HP 按声誉升序 |
| 弃置张数 | 在「弃低价值」前提下，可固定为 **能换则换满** 或 **只弃最低 1 张** —— 需在实现中二选一并写入代码注释；推荐 **尽可能多弃低价值牌** 以滤手牌，同时等量抓牌 |

### 策略说明

- 赤的核心是滤抽：AI 应优先弃掉低 HP、低声誉御魂/废牌
- 若手牌仅 1 张且为高价值，AI 可弃 0 张（仅保留伤害+1）

## ✅ 验收标准

### 功能验收

- [ ] 打出后伤害至少 +1
- [ ] 有手牌时可选任意子集弃置，并抓等量牌
- [ ] 可选择弃 0 张（仅伤害）
- [ ] 无手牌时仅伤害+1，不报错

### 边界情况

- [ ] 牌库张数 < 弃置张数：抓尽为止
- [ ] `onSelectCards` 返回非法 id：忽略非法项或按项目统一规则处理

### 强制中止 & AI接管

- [ ] 超时：伤害保留，不弃不抓
- [ ] AI：按价值序弃牌（若已实现）
