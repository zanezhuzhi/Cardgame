## 📜 效果原文

> 弃置你手上另一张生命不高于6的御魂，执行其效果两次。

---

## 🔍 效果分解

### 步骤拆解

```
触发时机: 打出时（立即触发）
├─ 步骤1: 检查手牌中是否有可弃置的御魂牌（HP ≤ 6 的御魂，排除轮入道自身）
│   ├─ 无可选牌 → 效果结束（无事发生）
│   └─ 有可选牌 → 继续
├─ 步骤2: 选择一张符合条件的御魂牌
├─ 步骤3: 将该牌从手牌弃置到弃牌堆
├─ 步骤4: 执行被弃置御魂的效果（第1次）
│   ├─ 若有交互选择 → 等待决策
│   └─ 完成后继续
├─ 步骤5: 执行被弃置御魂的效果（第2次）
│   ├─ 若有交互选择 → 等待决策（可与第1次不同）
│   └─ 完成后继续
└─ 结束: 轮入道效果完成
```

### 关键规则

| 规则点 | 说明 |
|--------|------|
| **"另一张"** | 选择的牌必须是手牌中的其他御魂牌，轮入道已打出，不在手牌 |
| **"生命不高于6"** | 包含HP=6的牌，即HP∈{1,2,3,4,5,6}均可选 |
| **"御魂"** | 只能选择subtype包含"御魂"的妖怪牌 |
| **"弃置"** | 牌从手牌移到弃牌堆（非超度，不进超度区）|
| **"执行两次"** | 完整执行两遍效果，而非数值翻倍 |
| **效果独立** | 每次执行都可独立做出不同选择 |

## 🎲 AI 行为决策

### 选择御魂牌的优先级

```typescript
function aiDecide_轮入道(hand: CardInstance[]): CardInstance | null {
  // 筛选可选御魂牌（HP ≤ 6，subtype包含御魂，排除自身）
  const validCards = hand.filter(card => 
    card.hp <= 6 && 
    card.subtype?.includes('御魂') &&
    card.name !== '轮入道'  // 打出后不在手牌，但保险起见
  );
  
  if (validCards.length === 0) return null;
  
  // 优先级评分（双倍效果下的收益）
  const scores = validCards.map(card => ({
    card,
    score: evaluateDoubleEffect(card)
  }));
  
  // 按分数排序，取最高
  scores.sort((a, b) => b.score - a.score);
  return scores[0].card;
}

  // 评分原则：
  // - 轮入道已消耗1张牌（弃置御魂），需要额外资源的牌评分应降低
  // - 无额外消耗、纯收益的牌评分更高
  // - 需要超度/弃置手牌的牌，会进一步加大资源消耗，评分较低
  
  const highValueCards = {
    // === 高收益（无额外消耗）===

    '狂骨': 6,        // 抓牌+2，伤害+2X，无额外消耗
    '日女巳时': 6,    // 选择两次，灵活组合，无额外消耗
    '鳴屋': 6,        // 伤害+4或+8，无额外消耗
    '针女': 6,        // 伤害+2，后续技能伤害+4，无额外消耗
    '返魂香': 5,      // 抓牌+2，伤害+2，妨害效果
    '雪幽魂': 5,      // 抓牌+2，妨害2次
    '天邪鬼青': 4,    // 抓牌+2或伤害+2
    '天邪鬼绿': 4,    // 退治两只，无额外消耗

    '天邪鬼黄': 4,    // 抓牌+4但要置顶2张
    '树妖': 4,        // 抓牌+4但要弃2（触发抓+2后净收益仍可观）
    '天邪鬼赤': 3,    // 伤害+2，但要弃置换牌×2
    '薙魂': 3,        // 抓+2弃2，需要已打3御魂才有鬼火收益
    '蚌精': 3,        // 超度2张手牌，抓牌+4（净亏2张牌换抓4）
    '唐纸伞妖': 3,    // 伤害+2，超度2张牌库顶（资源消耗中等）
    
    // === 低收益（高消耗或效果有限）===
    '骰子鬼': 2,      // 需超度2张手牌，退治2只（消耗大）

  };
```

## 🛠️ 实现备注

### 代码结构建议

```typescript
// shared/game/effects/YokaiEffects.ts

registerEffect('轮入道', async (ctx) => {
  const { player, triggerInteract, executeYokaiEffect } = ctx;
  
  // 1. 筛选可弃置的御魂
  const validCards = player.hand.filter(card => 
    card.hp <= 6 && 
    card.subtype?.includes('御魂')
  );
  
  if (validCards.length === 0) {
    return { success: true, message: '无可用御魂' };
  }
  
  // 2. 选择御魂
  const choice = await triggerInteract?.({
    type: 'selectCard',
    playerId: player.id,
    cards: validCards,
    prompt: '选择一张HP≤6的御魂弃置，执行其效果两次'
  });
  
  const selectedCard = validCards.find(c => c.instanceId === choice);
  if (!selectedCard) {
    return { success: false, error: '未选择有效御魂' };
  }
  
  // 3. 弃置御魂
  const cardIndex = player.hand.findIndex(c => c.instanceId === selectedCard.instanceId);
  const [discardedCard] = player.hand.splice(cardIndex, 1);
  player.discard.push(discardedCard);
  
  // 3.5 检查是否有【触】弃置触发效果
  await checkDiscardTrigger(discardedCard, ctx);
  
  // 4. 执行效果两次
  for (let i = 0; i < 2; i++) {
    await executeYokaiEffect(discardedCard.name, ctx);
  }
  
  return { success: true, message: `弃置${selectedCard.name}，执行效果两次` };
});
```

### 注意事项

1. **效果执行器**: 需要能够按名称调用其他御魂的效果处理器
2. **交互状态**: 每次执行效果时，若有选择，应独立询问
3. **减费叠加**: 涅槃之火执行两次，TempBuff的reduction应累加（-1 + -1 = -2）
4. **状态效果**: 网切等设置场地HP修正的效果，执行两次不叠加（同一状态覆盖）
5. **触发效果**: 弃置时的【触】效果应在执行打出效果之前触发
6. **地藏像**: 执行两次 = 超度1张（只超度地藏像本身）+ 获取2个式神；当前2式神则为1增1换
