## 📜 效果原文

> 本回合中，所有妖怪（包括手牌）生命-1，鬼王生命-2，但不会低于1点。

---

## 🔍 效果分解

### 步骤拆解

```
触发时机: 打出时（立即触发）
├─ 步骤1: 设置本回合临时状态
│   ├─ 妖怪HP修正: -1（适用于所有位置的妖怪牌）
│   └─ 鬼王HP修正: -2
├─ 步骤2: 状态生效范围
│   ├─ 场上游荡妖怪
│   ├─ 所有玩家手牌中的妖怪
│   ├─ 所有玩家牌库中的妖怪（若被抽出）
│   └─ 当前鬼王（及其遭遇区妖怪）
├─ 步骤3: HP最低值保护
│   └─ 任何妖怪/鬼王的HP不会低于1
└─ 结束: 回合结束时状态清除
```

### 关键规则

| 规则点 | 说明 |
|--------|------|
| **"本回合"** | 状态持续到当前玩家回合结束时清除（cleanup阶段） |
| **"所有妖怪"** | 包括场上游荡区、手牌、牌库、弃牌堆中的妖怪牌 |
| **"包括手牌"** | 明确说明手牌中的妖怪也受影响 |
| **"生命-1/-2"** | 是HP修正，而非造成伤害（不触发受伤效果） |
| **"不会低于1点"** | 即使计算后HP≤0，也显示为1 |
| **状态类效果** | 多次使用不叠加，只设置一次状态 |

## 🎲 AI 行为决策

### 使用时机评估

```typescript
function shouldUse_网切(ctx: EffectContext): boolean {
  const { player, gameState } = ctx;
  
  // 1. 检查游荡区是否有低HP妖怪可退治
  const wanderingYokai = gameState.field.wandering;
  const lowHpYokai = wanderingYokai.filter(y => y.hp <= 2);
  
  // 2. 检查鬼王HP
  const bossHp = gameState.field.boss?.hp || 0;
  const canKillBoss = bossHp <= player.damage + 2; // -2后可能击杀
  
  // 3. 评估使用价值
  const hasValue = 
    lowHpYokai.length > 0 ||  // 能帮助退治低HP妖怪
    canKillBoss ||            // 能帮助击杀鬼王
    bossHp <= 5;              // 鬼王HP较低，减2有意义
  
  return hasValue;
}
```

### AI 优先级

| 场景 | 建议 | 优先级 |
|------|------|--------|
| 游荡区有HP=2妖怪 | 使用后变HP=1，易退治 | 高 |
| 鬼王HP≤当前伤害+2 | 使用后可能击杀 | 高 |
| 手牌有高HP御魂 | 减1后更易管理 | 中 |
| 场上无低HP目标 | 收益较低 | 低 |

## ⚔️ 交互场景

### 场景1：正常使用 - 影响游荡区

```
游荡区: [心眼(HP5), 天邪鬼青(HP3)]
鬼王: 麒麟(HP8)

玩家打出: 网切

→ 设置本回合状态:
   - 妖怪HP修正: -1
   - 鬼王HP修正: -2

→ 实际HP显示:
   - 心眼: 5-1 = 4
   - 天邪鬼青: 3-1 = 2
   - 麒麟: 8-2 = 6

日志: "网切生效：本回合所有妖怪HP-1，鬼王HP-2"
```

### 场景2：最低值保护

```
游荡区: [招福达摩(HP1), 唐纸伞妖(HP2)]
鬼王: 麒麟(HP8)

玩家打出: 网切

→ HP计算:
   - 招福达摩: 1-1 = 0 → 保护为1
   - 唐纸伞妖: 2-1 = 1
   - 麒麟: 8-2 = 6

结果: 妖怪HP最低为1，鬼王正常减2
```

### 场景3：手牌妖怪受影响

```
玩家手牌: [心眼(HP5), 破势(HP6), 阴阳术·结界]

玩家打出: 网切

→ 手牌中妖怪HP显示:
   - 心眼: 5-1 = 4
   - 破势: 6-1 = 5
   - 阴阳术·结界: 不受影响（非妖怪）

注意: 若本回合打出心眼，其打出时HP为4（可能影响某些HP条件判定）
```

### 场景4：与其他玩家手牌的影响

```
多人游戏:
玩家A手牌: [镜姬(HP6)]
玩家B手牌: [狂骨(HP5)]

玩家A打出: 网切

→ 所有玩家手牌妖怪HP-1:
   - 玩家A的镜姬: 6-1 = 5
   - 玩家B的狂骨: 5-1 = 4

注意: 影响范围是全局的，不仅限于自己
```

### 场景5：轮入道弃置网切

```
玩家打出: 轮入道
玩家选择弃置: 网切

→ 执行网切效果（第1次）:
   - 设置状态：妖怪HP-1，鬼王HP-2
   
→ 执行网切效果（第2次）:
   - 再次设置同一状态（覆盖，不叠加）
   
结果: 仍然是妖怪HP-1，鬼王HP-2（状态不叠加）
```

### 场景6：回合结束后状态清除

```
玩家A回合:
├─ 打出网切 → 妖怪HP-1，鬼王HP-2
├─ 进行其他操作...
└─ 回合结束 → cleanup阶段清除状态

玩家B回合:
└─ 游荡区妖怪HP恢复原值（状态已清除）
```

## 🛠️ 实现备注

### 代码结构建议

```typescript
// shared/game/effects/YokaiEffects.ts

registerEffect('网切', async (ctx) => {
  const { player, gameState } = ctx;
  
  // 设置本回合临时状态（TempBuff）
  // 注意：这是场地级别的状态，而非玩家级别
  gameState.field.tempBuffs = gameState.field.tempBuffs || [];
  
  // 移除已存在的网切效果（避免重复）
  gameState.field.tempBuffs = gameState.field.tempBuffs.filter(
    buff => buff.type !== 'wangQieHpReduction'
  );
  
  // 添加新的网切效果
  gameState.field.tempBuffs.push({
    type: 'wangQieHpReduction',
    yokaiHpModifier: -1,
    bossHpModifier: -2,
    minHp: 1,
    expiresAt: 'endOfTurn',
    sourcePlayerId: player.id
  });
  
  return { success: true, message: '本回合所有妖怪HP-1，鬼王HP-2' };
});

// HP获取函数需要考虑修正
function getEffectiveHp(card: CardInstance, gameState: GameState): number {
  let hp = card.hp;
  
  // 检查场地临时状态
  const wangQieBuff = gameState.field.tempBuffs?.find(
    buff => buff.type === 'wangQieHpReduction'
  );
  
  if (wangQieBuff) {
    if (card.cardType === 'yokai') {
      hp += wangQieBuff.yokaiHpModifier;
    } else if (card.cardType === 'boss') {
      hp += wangQieBuff.bossHpModifier;
    }
    // 最低值保护
    hp = Math.max(hp, wangQieBuff.minHp);
  }
  
  return hp;
}
```

### 注意事项

1. **状态存储位置**: 应存储在 `gameState.field` 而非 `player`，因为影响全局
2. **HP显示**: 前端需要调用 `getEffectiveHp()` 显示修正后的HP
3. **退治判定**: 退治时使用修正后的HP值
4. **状态清除**: 在回合结束的cleanup阶段移除
5. **不叠加**: 多次使用只设置一次状态（覆盖而非累加）
