## 📜 效果原文

> 鬼火+1，抓牌+1，伤害+2，你左手边的玩家指定一个游荡妖怪或鬼王，本回合中你不能将其退治。

---

## 🔍 效果分解

### 步骤拆解

1. **鬼火+1**：当前玩家鬼火 +1（不超过上限）
2. **抓牌+1**：当前玩家从牌库抓 1 张牌
3. **伤害+2**：当前玩家本回合伤害 +2
4. **左手边玩家指定禁止目标**：
   - 当前玩家的左手边玩家（顺时针下一位）选择 1 个合法目标
   - 合法目标：游荡区的妖怪 **或** 当前鬼王
   - 本回合中，当前玩家不能退治该目标

### 关键规则

- **左手边玩家**：座位顺序中顺时针方向的下一位玩家
- **禁止退治（现行结算）**：场上规则为**击杀后默认退治**（不再二选一枚举退治/超度）。因此对**被限制的玩家**而言：
  - **不能**对该指定目标**分配伤害或完成可导致退治的击杀**（含鬼王攻击链、御魂“直接退治”类选择等）
  - **其他玩家**不受影响，仍可正常对该目标造成伤害与退治
  - **界面**：该游荡妖怪或鬼王上会显示**盾标**，**全场可见**；仅被限制的玩家无法将其作为分伤/攻击目标或选为退治目标
- **持续时间**：仅限本回合生效，回合结束后解除
- **单人模式**：无对手时，AI 代替左手边玩家执行选择

## 🎲 AI 行为决策

### AI 作为"左手边玩家"选择禁止目标

| 决策点 | AI策略 | 优先级说明 |
|--------|--------|-----------|
| 选择禁止目标 | 选择当前玩家最可能退治的目标 | HP最低的游荡妖怪 > 鬼王 |
| 目标HP相同时 | 选择声誉更高的目标 | 阻止对手获得更多声誉 |
| 仅剩鬼王时 | 选择鬼王 | 唯一合法目标 |

### AI 作为当前玩家使用此牌

| 决策点 | AI策略 | 说明 |
|--------|--------|------|
| 使用时机 | 手牌中有其他高伤害牌时优先使用 | 先获取资源再打伤害 |
| 被禁止目标后 | 攻击其他未被禁止的目标 | 调整攻击策略 |

## ⚔️ 交互场景

### 场景1：正常使用（多人模式）

```
当前状态:
- 玩家A 打出 镇墓兽
- 玩家A 左手边是 玩家B
- 游荡区: [心眼(HP5), 天邪鬼青(HP3)]
- 鬼王: 麒麟(HP8)

执行流程:
1. 玩家A 鬼火+1
2. 玩家A 抓牌+1
3. 玩家A 伤害+2
4. 玩家B 选择禁止目标 → 选择 天邪鬼青(HP3)
5. 本回合 玩家A 不能退治 天邪鬼青

结果:
- 玩家A 可以攻击心眼或麒麟
- 玩家A **不能**再对天邪鬼青分配伤害或将其击杀退治（击杀即默认退治）
```

### 场景2：单人模式（AI代选）

```
当前状态:
- 玩家 打出 镇墓兽
- 无其他玩家，AI代替左手边玩家
- 游荡区: [涂佛(HP5), 骰子鬼(HP4)]
- 鬼王: 石距(HP9)

执行流程:
1. 玩家 鬼火+1, 抓牌+1, 伤害+2
2. AI 选择禁止目标 → 选择 骰子鬼(HP4)（HP最低）
3. 本回合玩家不能退治骰子鬼

结果:
- 玩家可攻击涂佛或石距
```

### 场景3：游荡区为空

```
当前状态:
- 玩家A 打出 镇墓兽
- 游荡区: []（空）
- 鬼王: 麒麟(HP8)

执行流程:
1. 玩家A 鬼火+1, 抓牌+1, 伤害+2
2. 玩家B 只能选择鬼王 → 选择 麒麟
3. 本回合玩家A不能退治麒麟

结果:
- 玩家A本回合无法退治鬼王
```

### 场景4：被禁止后无法分伤击杀（与现行默认退治一致）

```
当前状态:
- 玩家A 打出 镇墓兽，玩家B 指定 天邪鬼青 为禁止目标
- 玩家A 本回合持有 3+ 伤害

执行流程:
1. 玩家A 点击天邪鬼青尝试分配伤害 → **不可用 / 服务端拒绝**
2. 玩家A 仅能对其他妖怪或鬼王输出伤害

结果:
- 玩家A 本回合不能由自己完成对该目标的击杀退治
```

### 场景5：其他玩家仍可处理同一目标

```
当前状态:
- 玩家A 被禁止退治 天邪鬼青
- 轮到玩家B 的回合且场上天邪鬼青仍存在

结果:
- 玩家B **可以**正常对天邪鬼青造成伤害并退治（不受玩家A 的禁止影响）
```

## 🛠️ 实现备注

### 涉及文件

- `shared/game/effects/YokaiEffects.ts` - 效果实现
- `shared/types/game.ts` - 可能需要添加 `prohibitedTargets` 字段到 PlayerState

### 关键函数

```typescript
registerEffect('镇墓兽', async (ctx) => {
  const { player, gameState, triggerInteract } = ctx;
  
  // 1. 鬼火+1
  player.ghostFire = Math.min(player.ghostFire + 1, player.maxGhostFire);
  
  // 2. 抓牌+1
  drawCards(player, 1);
  
  // 3. 伤害+2
  player.damage += 2;
  
  // 4. 左手边玩家选择禁止目标
  const leftPlayer = getLeftPlayer(gameState, player.id);
  const validTargets = [...gameState.field.wanderingYokai, gameState.field.currentBoss];
  
  const choice = await triggerInteract?.({
    type: 'selectTarget',
    playerId: leftPlayer.id,
    targets: validTargets,
    prompt: '选择一个目标，本回合对手不能退治它'
  });
  
  // 5. 记录禁止退治目标
  player.prohibitedTargets = [choice.targetId];
  
  return { success: true };
});
```

### 注意事项

- 需要在退治结算逻辑中检查 `prohibitedTargets`
- 回合结束时需清空 `prohibitedTargets`
- 单人模式需要AI代选逻辑
