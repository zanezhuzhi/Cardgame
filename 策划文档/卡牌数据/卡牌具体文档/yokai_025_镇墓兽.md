## 📜 效果原文

> 鬼火+1，抓牌+1，伤害+2，你左手边的玩家指定一个游荡妖怪或鬼王，本回合中你不能将其退治。

---

## 🔍 效果分解

### 步骤拆解

1. **鬼火+1**：当前玩家鬼火 +1（不超过上限）
2. **抓牌+1**：当前玩家从牌库抓 1 张牌
3. **伤害+2**：当前玩家本回合伤害 +2
4. **左手边玩家指定禁止目标**：
   - 当前玩家的左手边玩家（顺时针下一位）选择 1 个合法目标
   - 合法目标：游荡区的妖怪 **或** 当前鬼王
   - 本回合中，当前玩家不能退治该目标

### 关键规则

- **左手边玩家**：座位顺序中顺时针方向的下一位玩家
- **禁止退治 ≠ 禁止击杀**：
  - 被禁止的目标仍然可以受到伤害
  - 若HP降为0，回合结束时**自动超度**（进入超度区，不获得声誉）
  - 只是不能执行"退治"结算（不获得声誉、不触发退治效果）
- **持续时间**：仅限本回合生效，回合结束后解除
- **单人模式**：无对手时，AI 代替左手边玩家执行选择

## 🎲 AI 行为决策

### AI 作为"左手边玩家"选择禁止目标

| 决策点 | AI策略 | 优先级说明 |
|--------|--------|-----------|
| 选择禁止目标 | 选择当前玩家最可能退治的目标 | HP最低的游荡妖怪 > 鬼王 |
| 目标HP相同时 | 选择声誉更高的目标 | 阻止对手获得更多声誉 |
| 仅剩鬼王时 | 选择鬼王 | 唯一合法目标 |

### AI 作为当前玩家使用此牌

| 决策点 | AI策略 | 说明 |
|--------|--------|------|
| 使用时机 | 手牌中有其他高伤害牌时优先使用 | 先获取资源再打伤害 |
| 被禁止目标后 | 攻击其他未被禁止的目标 | 调整攻击策略 |

## ⚔️ 交互场景

### 场景1：正常使用（多人模式）

```
当前状态:
- 玩家A 打出 镇墓兽
- 玩家A 左手边是 玩家B
- 游荡区: [心眼(HP5), 天邪鬼青(HP3)]
- 鬼王: 麒麟(HP8)

执行流程:
1. 玩家A 鬼火+1
2. 玩家A 抓牌+1
3. 玩家A 伤害+2
4. 玩家B 选择禁止目标 → 选择 天邪鬼青(HP3)
5. 本回合 玩家A 不能退治 天邪鬼青

结果:
- 玩家A 可以攻击心眼或麒麟
- 玩家A 即使将天邪鬼青HP打到0，也不退治
```

### 场景2：单人模式（AI代选）

```
当前状态:
- 玩家 打出 镇墓兽
- 无其他玩家，AI代替左手边玩家
- 游荡区: [涂佛(HP5), 骰子鬼(HP4)]
- 鬼王: 石距(HP9)

执行流程:
1. 玩家 鬼火+1, 抓牌+1, 伤害+2
2. AI 选择禁止目标 → 选择 骰子鬼(HP4)（HP最低）
3. 本回合玩家不能退治骰子鬼

结果:
- 玩家可攻击涂佛或石距
```

### 场景3：游荡区为空

```
当前状态:
- 玩家A 打出 镇墓兽
- 游荡区: []（空）
- 鬼王: 麒麟(HP8)

执行流程:
1. 玩家A 鬼火+1, 抓牌+1, 伤害+2
2. 玩家B 只能选择鬼王 → 选择 麒麟
3. 本回合玩家A不能退治麒麟

结果:
- 玩家A本回合无法退治鬼王
```

### 场景4：被禁止目标HP=0时（关键边界）

```
当前状态:
- 玩家A 先打出其他牌，将 天邪鬼青(HP3) 打到 HP=0
- 玩家A 再打出 镇墓兽
- 玩家B 选择禁止目标 → 选择 天邪鬼青(HP=0)
- 玩家A 不选择退治任何目标，结束回合

执行流程:
1. 镇墓兽效果：鬼火+1, 抓牌+1, 伤害+2
2. 玩家B 指定 天邪鬼青 为禁止退治目标
3. 回合结束时，天邪鬼青(HP=0) 必须离场
4. 因为被禁止退治 → 自动超度

结果:
- 天邪鬼青 进入超度区（非弃牌堆）
- 玩家A 不获得声誉
- 玩家A 不触发退治相关效果
```

### 场景5：先镇墓兽后击杀

```
当前状态:
- 玩家A 打出 镇墓兽（伤害+2）
- 玩家B 选择禁止目标 → 选择 天邪鬼青(HP3)
- 玩家A 继续打牌，累计伤害达到3+
- 玩家A 选择攻击 天邪鬼青

执行流程:
1. 天邪鬼青 HP: 3 → 0
2. 回合结束时，天邪鬼青(HP=0) 被禁止退治
3. 自动超度

结果:
- 玩家A 无法获得 天邪鬼青 的声誉
- 玩家A 应优先攻击其他未被禁止的目标
```

## 🛠️ 实现备注

### 涉及文件

- `shared/game/effects/YokaiEffects.ts` - 效果实现
- `shared/types/game.ts` - 可能需要添加 `prohibitedTargets` 字段到 PlayerState

### 关键函数

```typescript
registerEffect('镇墓兽', async (ctx) => {
  const { player, gameState, triggerInteract } = ctx;
  
  // 1. 鬼火+1
  player.ghostFire = Math.min(player.ghostFire + 1, player.maxGhostFire);
  
  // 2. 抓牌+1
  drawCards(player, 1);
  
  // 3. 伤害+2
  player.damage += 2;
  
  // 4. 左手边玩家选择禁止目标
  const leftPlayer = getLeftPlayer(gameState, player.id);
  const validTargets = [...gameState.field.wanderingYokai, gameState.field.currentBoss];
  
  const choice = await triggerInteract?.({
    type: 'selectTarget',
    playerId: leftPlayer.id,
    targets: validTargets,
    prompt: '选择一个目标，本回合对手不能退治它'
  });
  
  // 5. 记录禁止退治目标
  player.prohibitedTargets = [choice.targetId];
  
  return { success: true };
});
```

### 注意事项

- 需要在退治结算逻辑中检查 `prohibitedTargets`
- 回合结束时需清空 `prohibitedTargets`
- 单人模式需要AI代选逻辑
