## 📜 效果原文

> 伤害+1。本回合中，每当你使用式神技能时，伤害+2。

---

## 🔍 效果分解

### 步骤拆解

1. **伤害+1**：立即生效，当前玩家本回合伤害 +1
2. **注册回合内持续效果**：本回合中，每当使用式神技能时，额外伤害+2

### 关键规则

- **触发时机**：每次使用式神技能时触发，可多次触发
- **伤害+2**：是额外伤害，叠加到本回合伤害池
- **持续时间**：仅本回合有效，回合结束后解除
- **式神技能定义**：式神卡牌上标注的主动技能（需消耗鬼火使用）
- **多张针女**：效果可叠加，打出2张针女后，每次使用技能伤害+4

### 与式神技能的交互

| 式神技能类型 | 是否触发针女 |
|-------------|:-----------:|
| 主动技能（消耗鬼火） | ✅ 触发 |
| 被动技能（自动生效） | ❌ 不触发 |
| 退治后触发的效果 | ❌ 不触发 |

## ⚔️ 交互场景

### 场景1：正常使用（单次技能）

```
当前状态:
- 玩家 打出 针女
- 玩家拥有式神: 大天狗（技能：伤害+3）
- 玩家鬼火: 3

执行流程:
1. 针女效果：伤害+1
2. 玩家使用 大天狗技能（消耗鬼火）
3. 触发针女效果：伤害+2
4. 大天狗技能本身：伤害+3

结果:
- 本回合总伤害: 1 + 2 + 3 = 6
```

### 场景2：多次使用技能

```
当前状态:
- 玩家 打出 针女
- 玩家拥有式神: 茨木童子（技能：伤害+2）、酒吞童子（技能：伤害+2）
- 玩家鬼火: 6（足够使用两个技能）

执行流程:
1. 针女效果：伤害+1
2. 玩家使用 茨木童子技能 → 针女触发：伤害+2
3. 玩家使用 酒吞童子技能 → 针女触发：伤害+2
4. 两个技能本身各+2

结果:
- 本回合总伤害: 1 + 2 + 2 + 2 + 2 = 9
```

### 场景3：多张针女叠加

```
当前状态:
- 玩家 打出 2张针女
- 玩家拥有式神: 大天狗（技能：伤害+3）

执行流程:
1. 第1张针女：伤害+1
2. 第2张针女：伤害+1
3. 玩家使用 大天狗技能
4. 第1张针女触发：伤害+2
5. 第2张针女触发：伤害+2
6. 大天狗技能本身：伤害+3

结果:
- 本回合总伤害: 1 + 1 + 2 + 2 + 3 = 9
```

### 场景4：技能后打出针女（无收益）

```
当前状态:
- 玩家先使用 大天狗技能
- 然后打出 针女

执行流程:
1. 大天狗技能：伤害+3（针女还未打出，无触发）
2. 针女效果：伤害+1
3. 本回合不再使用技能

结果:
- 本回合总伤害: 3 + 1 = 4
- 针女的+2触发效果被浪费
```

### 场景5：无式神时使用

```
当前状态:
- 玩家 打出 针女
- 玩家没有式神（或式神没有可用技能）

执行流程:
1. 针女效果：伤害+1
2. 无技能可用，+2效果无法触发

结果:
- 本回合伤害: 1
- 针女仍有基础收益
```

## 🛠️ 实现备注

### 涉及文件

- `shared/game/effects/YokaiEffects.ts` - 效果实现
- `shared/types/game.ts` - 可能需要添加 `onSkillUsed` 回调或 `tempBuffs` 数组

### 关键函数

```typescript
registerEffect('针女', async (ctx) => {
  const { player, gameState } = ctx;
  
  // 1. 伤害+1
  player.damage += 1;
  
  // 2. 注册本回合持续效果
  player.tempBuffs.push({
    type: 'onSkillUsed',
    effect: () => {
      player.damage += 2;
      ctx.addLog?.(`⚡ 针女触发：伤害+2`);
    },
    duration: 'turn' // 本回合结束后自动移除
  });
  
  return { success: true, message: '伤害+1' };
});
```

### 注意事项

- 需要在式神技能执行逻辑中检查并触发 `onSkillUsed` 回调
- 回合结束时需清理 `tempBuffs` 中 `duration: 'turn'` 的效果
- 多张针女需要独立注册回调，确保各自触发
