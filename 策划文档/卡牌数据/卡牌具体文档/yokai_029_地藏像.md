## 📜 效果原文

> 超度此牌。获取1个式神（式神上限为3个）。

---

## 🔍 效果分解

### ⚠️ 特殊机制：二次确认

> **地藏像没有"打出效果"**（不像其他牌有伤害+X等即时收益），打出即超度。
> 为防止误操作，点击打出后需**弹出二次确认**。

### 步骤拆解

1. **点击打出** → 弹出确认对话框：「确定打出地藏像？此牌将被超度」
2. **玩家确认** → 地藏像进入超度区（非弃牌堆）
3. **检查上限** → 判断当前式神数量，决定进入「获取」或「置换」流程
4. **二选一流程**（标准式神获取）→ 从式神牌库抽取顶部2张，选择1张
5. **如果是置换** → 选择要替换的式神，或放弃获取

### 关键规则

- **超度而非弃置**：地藏像使用后进入超度区，不进入弃牌堆
- **式神上限**：每个玩家最多**保留**3个式神
- **替换流程**：如果已有3个式神，获取新式神时触发**替换选择**
  - 玩家可选择替换掉1个现有式神（被替换的式神回到牌库）
  - 也可选择**不替换**（放弃获取新式神）
- **式神来源**：从公共式神牌库中随机/选择获取
- **高声誉**：charm = 2，退治此卡可获得2点声誉

### 式神获取规则

| 条件 | 行为 |
|------|------|
| 当前式神 < 3 | 超度地藏像 + 二选一获取式神 |
| 当前式神 = 3 | 超度地藏像 + **触发置换流程**（可选择替换或放弃） |

> ⚠️ **理论边界**：式神牌库为空时，仅超度不获取。但实际游戏中**不可能出现**此情况。

|:------:|----------|

| 2 | 资源型式神（抓牌/鬼火） |

## 🛠️ 实现备注

### 涉及文件

- `shared/game/effects/YokaiEffects.ts` - 效果实现
- `shared/types/game.ts` - 式神相关类型

### 关键函数

```typescript
registerEffect('地藏像', async (ctx) => {
  const { player, gameState, card, triggerInteract } = ctx;
  
  // 1. 超度此牌（从手牌移到超度区）
  player.exiled.push(card);
  
  // 2. 检查式神牌库
  if (gameState.shikigamiDeck.length === 0) {
    ctx.addLog?.(`⚠️ 式神牌库为空，无法获取`);
    return { success: true, message: '超度完成，无可用式神' };
  }
  
  // 3. 判断当前式神数量，决定流程
  const isReplace = player.shikigami.length >= 3;
  
  // 4. 二选一流程：从牌库顶抽取2张
  const drawCount = Math.min(2, gameState.shikigamiDeck.length);
  const drawnShikigami = gameState.shikigamiDeck.splice(0, drawCount);
  
  // 5. 玩家选择1张
  const selectChoice = await triggerInteract?.({
    type: 'selectShikigami',
    playerId: player.id,
    options: drawnShikigami,
    prompt: '从2张式神中选择1张'
  });
  
  const selectedIndex = selectChoice?.selectedIndex ?? 0;
  const selectedShikigami = drawnShikigami[selectedIndex];
  const unselectedShikigami = drawnShikigami.filter((_, i) => i !== selectedIndex);
  
  // 6. 未选中的放回牌库底部
  gameState.shikigamiDeck.push(...unselectedShikigami);
  
  // 7. 如果是置换流程，选择要替换的式神
  if (isReplace) {
    const replaceChoice = await triggerInteract?.({
      type: 'replaceShikigami',
      playerId: player.id,
      currentShikigami: player.shikigami,
      newShikigami: selectedShikigami,
      prompt: `选择要替换的式神，或取消`
    });
    
    if (replaceChoice?.cancel || replaceChoice?.replaceId === null) {
      // 放弃获取，选中的式神也放回牌库底部
      gameState.shikigamiDeck.push(selectedShikigami);
      ctx.addLog?.(`❌ 放弃获取 ${selectedShikigami.name}`);
      return { success: true, message: '超度完成，放弃置换' };
    }
    
    // 执行替换
    const replaceIndex = player.shikigami.findIndex(s => s.id === replaceChoice.replaceId);
    if (replaceIndex !== -1) {
      const replacedShikigami = player.shikigami.splice(replaceIndex, 1)[0];
      gameState.shikigamiDeck.push(replacedShikigami); // 被替换的回到牌库底部
      ctx.addLog?.(`🔄 ${replacedShikigami.name} 回到式神牌库底部`);
    }
  }
  
  // 8. 获取新式神
  player.shikigami.push(selectedShikigami);
  ctx.addLog?.(`✨ 获取式神: ${selectedShikigami.name}`);
  
  return { success: true, message: '超度完成，获取式神' };
});
```

### 注意事项

- 超度与弃置不同，需要正确处理卡牌去向
- **二选一流程**：从牌库顶抽取2张，选择1张，未选中的放回牌库底部
- **置换流程**：式神已满时需要两步交互（二选一 → 选择替换目标/取消）
- 被替换的式神、未选中的式神均放回式神牌库**底部**
