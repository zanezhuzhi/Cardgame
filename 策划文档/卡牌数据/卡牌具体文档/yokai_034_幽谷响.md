## 📜 效果原文

> 每名对手展示牌库顶牌，你可以使用至多3张以此法展示的非鬼王的效果，然后，将它们置于拥有者的弃牌区。

## 🔍 效果分解

### 步骤拆解

1. **触发时机**：玩家打出幽谷响作为御魂
2. **展示牌库顶**：每名对手展示自己牌库顶的1张牌
3. **筛选可用牌**：排除鬼王卡，剩余为「可选池」
4. **选择使用**：玩家从「可选池」中选择 0~3 张牌使用其效果
5. **结算效果**：按选择顺序依次执行所选牌的效果
6. **归还弃牌区**：所有展示的牌（无论是否使用）置于各自拥有者的弃牌区

### 关键规则

| 规则 | 说明 |
|------|------|
| 展示数量 | 每名对手展示 **1张** 牌库顶牌 |
| 排除鬼王 | 鬼王卡不可被选择使用 |
| 使用上限 | 最多使用 **3张** 展示牌的效果 |
| 使用是可选的 | 可以选择 0 张（不使用任何效果） |
| 效果归属 | 使用的是「牌的效果」，但执行者是当前玩家（幽谷响使用者） |
| 归还位置 | 所有展示的牌最终进入 **拥有者的弃牌区**（不是使用者的） |

### 效果执行细节

| 情况 | 处理方式 |
|------|----------|
| 对手牌库为空 | 该对手跳过展示，不参与此效果 |
| 展示的是鬼王 | 展示后直接进入拥有者弃牌区，不可被选择 |
| 展示牌有选择效果 | 由幽谷响使用者做出选择 |
| 展示牌涉及"你" | "你"指幽谷响使用者 |
| 多人模式 | 更多对手 = 更多展示牌 = 更多选择 |

### 数值分析

| 对手数量 | 展示牌数 | 最多可用 | 场景价值 |
|:--------:|:--------:|:--------:|----------|
| 1人 | 1张 | 1张 | 效果有限 |
| 2人 | 2张 | 2张 | 标准收益 |
| 3人 | 3张 | 3张 | 刚好满配 |
| 4-5人 | 4-5张 | 3张 | 可挑选最优 |

## 🎲 AI 行为决策

### 选择使用哪些牌 (L1 规则策略)

```
输入: 展示的非鬼王牌列表
输出: 要使用的牌列表 (0~3张)

决策逻辑:
1. 按效果价值对展示牌评分:
   - 伤害类: 伤害值 × 2
   - 抓牌类: 抓牌数 × 1.5
   - 鬼火类: 鬼火值 × 1
   - 声誉类: 声誉值 × 0.5
   - 妨害类: 0 (对自己无益)
   
2. 按评分降序排序
3. 选择前 min(3, 展示牌数) 张牌
4. 若所有牌评分 ≤ 0，则不选择任何牌
```

### 效果内的二次选择

```
若被选牌有内部选择（如二选一）:
- 按标准 AI 策略执行该牌效果
- 例: 展示的「骰子鬼」触发时，AI 选择伤害+2
```

| 决策点 | AI策略 | 优先级说明 |
|--------|--------|-----------|
| `onSelectCards` | 按效果价值选择最优的3张 | 伤害 > 抓牌 > 鬼火 |
| 内部选择 | 复用对应卡牌的 AI 逻辑 | 参考各卡牌文档 |

### 交互流程

```typescript
// 伪代码
registerEffect('幽谷响', async (ctx) => {
  const { player, gameState, triggerInteract, executeCardEffect } = ctx;
  
  // 1. 收集所有对手的牌库顶牌
  const revealedCards: { card: CardInstance, owner: PlayerState }[] = [];
  for (const opponent of getOpponents(player, gameState)) {
    if (opponent.deck.length > 0) {
      const topCard = opponent.deck[0];
      revealedCards.push({ card: topCard, owner: opponent });
    }
  }
  
  // 2. 展示并筛选非鬼王
  const selectableCards = revealedCards.filter(r => r.card.type !== 'boss');
  
  // 3. 让玩家选择使用哪些 (0~3张)
  const selected = await triggerInteract?.({
    type: 'selectCards',
    playerId: player.id,
    cards: selectableCards.map(r => r.card),
    min: 0,
    max: Math.min(3, selectableCards.length),
    prompt: `选择要使用的效果 (最多3张)`
  }) ?? [];
  
  // 4. 执行所选牌的效果
  for (const card of selected) {
    await executeCardEffect(card, { ...ctx, triggeringCard: card });
  }
  
  // 5. 所有展示的牌归还拥有者弃牌区
  for (const { card, owner } of revealedCards) {
    owner.deck.shift(); // 移除牌库顶
    owner.discard.push(card);
  }
  
  return { 
    success: true, 
    message: `展示了 ${revealedCards.length} 张牌，使用了 ${selected.length} 张效果` 
  };
});
```

### 注意事项

1. **效果归属**：执行展示牌效果时，`player` 仍是幽谷响使用者
2. **牌的移动**：执行完效果后，牌从牌库顶移至拥有者弃牌区（不是使用者的）
3. **递归效果**：若展示的牌也是幽谷响，理论上可以再次触发（需防止无限循环）
4. **日志格式**：`🔮 ${player.name} 使用幽谷响，展示了 [${cardNames}]，使用了 [${usedNames}] 的效果`
5. **AI选择**：需要为每种牌类型设定评分规则，优先选择高价值效果
