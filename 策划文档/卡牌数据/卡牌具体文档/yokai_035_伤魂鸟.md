## 📜 效果原文

> 超度X张手牌。然后伤害+2X。

---

## 🔍 效果分解

### 步骤拆解

1. **触发时机**：玩家打出伤魂鸟作为御魂
2. **选择超度数量**：玩家选择要超度的手牌数量 X（0 ≤ X ≤ 手牌数）
3. **执行超度**：玩家选择 X 张手牌，将它们移至超度区
4. **结算伤害**：伤害 += 2 × X

### 关键规则

| 规则 | 说明 |
|------|------|
| X 的范围 | 0 ≤ X ≤ 当前手牌数量（不含伤魂鸟本身，因为已经打出） |
| X = 0 | 可以选择不超度任何牌，此时伤害 +0（等于无效果） |
| 超度是「可选」 | 玩家主动决定超度数量，非强制 |
| 两步选择 | 先选超度数量 X，再选择具体 X 张牌 |
| 超度目标限制 | 只能超度手牌，不能超度场上其他牌 |

### 数值分析

| X 值 | 伤害增益 | 手牌代价 | 收益评估 |
|:----:|:--------:|:--------:|----------|
| 0 | +0 | 0张 | 无效果，仅获得御魂 |
| 1 | +2 | 1张 | 低收益，保守打法 |
| 2 | +4 | 2张 | 中等收益 |
| 3 | +6 | 3张 | 高伤害，手牌代价大 |
| 4+ | +8+ | 4+张 | 极端策略，清空手牌 |

## 🎲 AI 行为决策

### 超度数量决策 (L1 规则策略)

```
输入: 当前手牌列表 (不含伤魂鸟)
输出: 要超度的牌数量 X

决策逻辑:
1. 计算手牌中"低价值牌"数量 (HP ≤ 3 的妖怪)
2. 计算当前伤害缺口 = 目标HP - 当前伤害
3. X = min(低价值牌数量, ceil(伤害缺口 / 2), 手牌数)
4. 若 X = 0 但伤害缺口 > 0，则 X = 1 (至少超度1张)
```

### 选择超度目标 (L1 规则策略)

```
输入: 需要超度的数量 X，当前手牌列表
输出: X 张要超度的牌

决策逻辑:
1. 按优先级排序手牌:
   - 优先超度: 恶评卡 > HP低的妖怪 > 阴阳术 > HP高的妖怪
2. 选择前 X 张低优先级的牌
```

| 决策点 | AI策略 | 优先级说明 |
|--------|--------|-----------|
| `onChooseX` | 根据伤害缺口和低价值牌数量计算 | 保留关键牌 |
| `onSelectCards` | 选择价值最低的牌 | 恶评 > HP低 > 阴阳术 |

## ⚔️ 交互场景

### 场景1：标准超度

```
初始状态:
玩家手牌: [伤魂鸟, 赤舌(HP2), 天邪鬼青(HP2), 灯笼鬼(HP3)]
当前伤害: 0

执行:
1. 打出伤魂鸟
2. 选择超度数量: X = 2
3. 选择超度目标: [赤舌, 天邪鬼青]
4. 赤舌、天邪鬼青 → 超度区
5. 伤害 += 2 × 2 = 4

结果:
玩家手牌: [灯笼鬼]
玩家超度区: [赤舌, 天邪鬼青]
当前伤害: 4
```

### 场景2：不超度任何牌

```
初始状态:
玩家手牌: [伤魂鸟, 心眼(HP5), 破势(HP6)]
当前伤害: 5

执行:
1. 打出伤魂鸟
2. 选择超度数量: X = 0 (保留高价值牌)
3. 无需选择超度目标
4. 伤害 += 2 × 0 = 0

结果:
玩家手牌: [心眼, 破势]
当前伤害: 5 (不变)
```

### 场景3：全部超度（All-In）

```
初始状态:
玩家手牌: [伤魂鸟, A, B, C, D] (4张其他牌)
当前伤害: 0
目标妖怪HP: 8

执行:
1. 打出伤魂鸟
2. 选择超度数量: X = 4 (全部超度)
3. 选择超度目标: [A, B, C, D]
4. 伤害 += 2 × 4 = 8

结果:
玩家手牌: [] (空)
玩家超度区: [A, B, C, D]
当前伤害: 8 (刚好够退治HP8的妖怪)
```

### 场景4：边界情况 - 手牌为空

```
初始状态:
玩家手牌: [伤魂鸟] (仅1张)
当前伤害: 0

执行:
1. 打出伤魂鸟
2. 打出后手牌为空，X 只能 = 0
3. 伤害 += 0

结果:
玩家手牌: []
当前伤害: 0
```

### 交互流程

```typescript
// 伪代码
registerEffect('伤魂鸟', async (ctx) => {
  const { player, triggerInteract } = ctx;
  
  // 1. 选择超度数量 X
  const maxX = player.hand.length; // 打出后的手牌数
  const x = await triggerInteract?.({
    type: 'chooseNumber',
    playerId: player.id,
    min: 0,
    max: maxX,
    prompt: `选择超度手牌数量 (0-${maxX})`
  }) ?? 0;
  
  // 2. 如果 X > 0，选择具体哪些牌
  if (x > 0) {
    const selected = await triggerInteract?.({
      type: 'selectCards',
      playerId: player.id,
      source: 'hand',
      count: x,
      prompt: `选择 ${x} 张手牌超度`
    });
    
    // 3. 执行超度
    for (const card of selected) {
      removeFromHand(player, card);
      player.exiled.push(card);
    }
  }
  
  // 4. 伤害结算
  player.damage += 2 * x;
  
  return { 
    success: true, 
    message: x > 0 ? `超度了 ${x} 张牌，伤害 +${2 * x}` : '未超度任何牌' 
  };
});
```

### 注意事项

1. **两阶段选择**：先选数量 X，再选 X 张牌，不能合并为一步
2. **AI 超度优先级**：恶评 > HP低 > 阴阳术 > HP高
3. **与轮入道的联动**：如果伤魂鸟被轮入道弃置，仍然可以执行效果（参考 yokai_019_轮入道.md 场景3）
4. **日志格式**：`🔥 ${player.name} 超度了 ${x} 张牌，伤害 +${2*x}`
