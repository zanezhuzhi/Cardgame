## 🔍 效果分解

### 步骤拆解

1. **触发时机**：玩家打出阴摩罗作为御魂
2. **选择目标**：从自己弃牌区选择 2 张 HP < 6 的牌
3. **使用效果**：依次使用所选牌的效果
4. **延迟归还**：在**回合结束时**，将这2张牌返回牌库底

### 关键规则

| 规则 | 说明 |
|------|------|
| HP 限制 | 只能选择 HP < 6 的牌（即 HP ≤ 5） |
| 选择数量 | **恰好 2 张**（若弃牌区不足则选择全部可用牌） |
| 效果归属 | 使用牌的效果，执行者是阴摩罗使用者 |
| 归还时机 | **回合结束时**（清算阶段后）返回牌库底 |
| 归还位置 | 牌库**底部**（不是顶部） |
| 牌的来源 | **自己的**弃牌区，不能选对手的 |

### HP 筛选范围

| 性别 | 女 |
| HP | 可选 | 典型代表 |
|:--:|:----:|----------|
| 1 | ✅ | 招福达摩 |
| 2 | ✅ | 赤舌、天邪鬼系列 |
| 3 | ✅ | 灯笼鬼、树妖、蚌精 |
| 4 | ✅ | 骰子鬼、轮入道、网切 |
| 5 | ✅ | 心眼、狂骨、针女 |
| 6+ | ❌ | 镜姬、破势、阴摩罗本身 |

### 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 弃牌区无 HP<6 的牌 | 效果无法执行，跳过选择 |
| 弃牌区仅 1 张 HP<6 | 选择该 1 张使用 |
| 选中的牌有选择效果 | 由阴摩罗使用者做出选择 |
| 回合中牌被其他效果移动 | 仍按原规则在回合结束时归还 |

## 🎲 AI 行为决策

### 选择弃牌区的牌 (L1 规则策略)

```
输入: 弃牌区中 HP < 6 的牌列表
输出: 要使用的2张牌

决策逻辑:
1. 按效果价值评分:
   - 伤害类: 伤害值 × 2
   - 抓牌类: 抓牌数 × 1.5
   - 鬼火类: 鬼火值 × 1
   - 声誉类: 0 (回合结束归还，无法获得)
   - 妨害类: 0 (对自己无益)
   
2. 按评分降序排序
3. 选择前 min(2, 可用牌数) 张
```

### 效果内的二次选择

```
若被选牌有内部选择（如二选一）:
- 按标准 AI 策略执行该牌效果
- 例: 选中「骰子鬼」触发时，AI 选择伤害+2
```

| 决策点 | AI策略 | 优先级说明 |
|--------|--------|-----------|
| `onSelectCards` | 选择效果价值最高的2张 | 伤害 > 抓牌 > 鬼火 |
| 内部选择 | 复用对应卡牌的 AI 逻辑 | 参考各卡牌文档 |

## 🛠️ 实现备注

### 涉及文件

- `shared/game/effects/YokaiEffects.ts` - 注册阴摩罗效果
- `shared/game/effects/YokaiEffects.test.ts` - TDD测试用例
- `client/src/components/SelectCardsDialog.vue` - 选牌 UI (复用)

### 交互流程

```typescript
// 伪代码
registerEffect('阴摩罗', async (ctx) => {
  const { player, gameState, triggerInteract, executeCardEffect, registerEndOfTurnEffect } = ctx;
  
  // 1. 筛选弃牌区 HP < 6 的牌
  const eligibleCards = player.discard.filter(card => card.hp < 6);
  
  if (eligibleCards.length === 0) {
    return { success: true, message: '弃牌区无可用牌' };
  }
  
  // 2. 选择 2 张（或全部可用牌）
  const maxSelect = Math.min(2, eligibleCards.length);
  const selected = await triggerInteract?.({
    type: 'selectCards',
    playerId: player.id,
    cards: eligibleCards,
    min: maxSelect, // 必须选满
    max: maxSelect,
    prompt: `选择 ${maxSelect} 张牌使用`
  }) ?? eligibleCards.slice(0, maxSelect);
  
  // 3. 从弃牌区移除（临时）
  for (const card of selected) {
    const idx = player.discard.findIndex(c => c.instanceId === card.instanceId);
    if (idx !== -1) player.discard.splice(idx, 1);
  }
  
  // 4. 执行效果
  for (const card of selected) {
    await executeCardEffect(card, { ...ctx, triggeringCard: card });
  }
  
  // 5. 注册回合结束归还
  registerEndOfTurnEffect(() => {
    for (const card of selected) {
      player.deck.push(card); // 放到牌库底
    }
    ctx.addLog(`🔄 ${selected.map(c => c.name).join('、')} 返回牌库底`);
  });
  
  return { 
    success: true, 
    message: `使用了 ${selected.map(c => c.name).join('、')} 的效果` 
  };
});
```

### 注意事项

1. **HP 判断**：严格 `hp < 6`（即 HP ≤ 5），HP=6 的牌不可选
2. **回合结束归还**：需要实现 `registerEndOfTurnEffect` 机制
3. **牌的临时存放**：执行效果期间，牌既不在弃牌区也不在手牌
4. **递归效果**：若选中的牌是轮入道，其效果中弃置的牌**不会**被阴摩罗归还
5. **日志格式**：
   - 使用时：`🌙 ${player.name} 使用阴摩罗，选择了 [${cardNames}]`
   - 归还时：`🔄 [${cardNames}] 返回牌库底`
