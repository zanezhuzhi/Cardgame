## 📜 效果原文

> 抓牌+2，鬼火+1。受到妨害和鬼王来袭时，你可以展示本卡而不受影响。

---

## 🔍 效果分解

### 步骤拆解

**御魂效果（打出时）**：
1. **抓牌+2**：从牌库顶抓取2张牌
2. **鬼火+1**：获得1点鬼火（上限5）

**手牌效果（被动防御）**：
3. **展示防御**：当受到妨害或鬼王来袭时，可展示本卡免疫该效果

### 关键规则

| 规则 | 说明 |
|------|------|
| 御魂效果 | 打出时立即执行：抓牌+2，鬼火+1 |
| 展示防御 | **在手牌中**时可被动触发，防御妨害/来袭 |
| 展示条件 | 受到妨害或鬼王来袭效果时 |
| 展示结果 | 展示后**不受该效果影响**，牌**留在手中** |
| 可选触发 | "你可以展示" = 玩家**选择**是否展示 |
| 不消耗 | 展示不等于打出，展示后牌仍在手牌中 |

### 可防御的效果类型

| 效果来源 | 类型 | 示例 |
|----------|------|------|
| **妨害** | 其他玩家的妨害效果 | 魍魉之匣（声誉+2给对手恶评）、镇墓兽（禁止退治目标） |
| **鬼王来袭** | 鬼王翻出时的全体效果 | 各鬼王的【来袭】效果 |

### 不可防御的效果

| 情况 | 说明 |
|------|------|
| 非妨害的御魂效果 | 正常的御魂效果（如伤害、抓牌）不是妨害 |
| 已打出的青女房 | 打出后不在手牌，无法展示 |
| 式神技能 | 式神技能不属于「妨害」或「来袭」 |

## 🎲 AI 行为决策

### 是否展示防御 (L1 规则策略)

```
输入: 当前受到的妨害/来袭效果
输出: 是否展示青女房

决策逻辑:
1. 评估效果负面程度:
   - 高负面: 获得恶评(-1/-2声誉)、弃置手牌、强制超度
   - 中负面: 禁止退治、鬼火-1
   - 低负面: 其他轻微效果
   
2. 决策规则:
   - 高负面效果 → 立即展示防御
   - 中负面效果 → 展示防御
   - 低负面效果 → 评估手牌价值后决定

3. 保守策略:
   - 默认选择展示（除非效果极轻微）
```

| 决策点 | AI策略 | 优先级说明 |
|--------|--------|-----------|
| `onRevealChoice` | 评估效果负面程度后决定 | 恶评/弃牌 > 禁止退治 > 轻微效果 |

### 交互流程

```typescript
// 御魂效果
registerEffect('青女房', async (ctx) => {
  const { player } = ctx;
  
  // 1. 抓牌+2
  drawCards(player, 2);
  
  // 2. 鬼火+1
  player.ghostFire = Math.min(player.ghostFire + 1, 5);
  
  return { success: true, message: '抓牌+2，鬼火+1' };
});

// 展示防御机制（在妨害/来袭触发时调用）
async function checkRevealDefense(
  target: PlayerState, 
  effectType: 'harassment' | 'bossRaid'
): Promise<boolean> {
  // 检查手牌中是否有青女房
  const qingnvfang = target.hand.find(c => c.name === '青女房');
  if (!qingnvfang) return false;
  
  // 询问是否展示
  const choice = await triggerInteract?.({
    type: 'revealDefense',
    playerId: target.id,
    card: qingnvfang,
    effectType,
    prompt: `是否展示青女房抵挡${effectType === 'harassment' ? '妨害' : '来袭'}？`
  });
  
  if (choice === true) {
    // 展示动画
    addLog(`🛡️ ${target.name} 展示青女房，免疫${effectType === 'harassment' ? '妨害' : '来袭'}`);
    return true; // 免疫效果
  }
  
  return false; // 不展示，正常受到效果
}
```

### 注意事项

1. **展示 ≠ 打出**：展示后牌仍在手中，不进入已打出区
2. **触发时机**：在妨害/来袭效果**结算前**询问，选择展示则跳过该效果
3. **多张青女房**：只需展示1张即可免疫，其他青女房保留
4. **妨害定义**：需要在代码中标记哪些效果属于「妨害」类型
5. **日志格式**：`🛡️ ${player.name} 展示青女房，免疫${effectType}`

### 技术依赖

需要实现「妨害」和「来袭」效果的分类标记系统：

```typescript
interface EffectContext {
  effectType?: 'normal' | 'harassment' | 'bossRaid';
  // ...
}

// 在触发妨害效果时
async function triggerHarassment(source: PlayerState, targets: PlayerState[], effect: Function) {
  for (const target of targets) {
    // 检查是否有防御手段
    const defended = await checkRevealDefense(target, 'harassment');
    if (!defended) {
      await effect(target);
    }
  }
}
```
