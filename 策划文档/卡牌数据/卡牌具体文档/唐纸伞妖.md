# 唐纸伞妖

> 妖怪/御魂卡

---

## 基础信息

| 属性 | 值 |
|------|---|
| ID | yokai_002 |
| 名称 | 唐纸伞妖 |
| HP | 2 |
| 声誉 | 0 |
| 类型 | 御魂 |
| 效果 | 伤害+1，查看你牌库顶牌，你可以超度它。 |
| 多人标记 | 1（单人/多人都有效） |

---

## 效果分解

1. **伤害+1** - 立即增加当前回合的伤害值
2. **查看牌库顶牌** - 展示牌库最顶部的1张牌给玩家
3. **可选超度** - 玩家决定是否将该牌移动到超度区
   - 选择"超度"：牌库顶牌 → 超度区
   - 选择"不超度"：牌库顶牌保持原位

---

## 单人模式实现

### 逻辑流程

```
1. 触发御魂效果
2. damage += 1
3. 获取 deck[0]（牌库顶牌）
4. 弹窗展示卡牌，询问"是否超度？"
5. if (用户选择超度) {
     deck.shift() → salvageZone.push()
   }
6. 效果结束
```

### 涉及模块

- `client/src/game/SinglePlayerGame.ts` - 单人游戏逻辑
- `client/src/App.vue` - UI弹窗

### 风险点

- 牌库为空时需要特殊处理
- 弹窗需要阻塞后续流程，等待用户选择

## 多人模式实现

### 逻辑流程

```
1. 触发御魂效果
2. state.players[playerId].damage += 1
3. 获取 state.players[playerId].deck[0]
4. 设置等待状态 pendingAction = { type: 'salvageChoice', card: deckTopCard }
5. 向客户端发送选择请求
6. 等待客户端响应
7. if (用户选择超度) {
     deck.shift() → salvageZone.push()
   }
8. 清除 pendingAction
9. 效果结束，继续游戏流程
```

### 涉及模块

- `server/src/game/MultiplayerGame.ts` - 服务端游戏逻辑
- `server/src/socket/SocketServer.ts` - Socket通信
- `client/src/App.vue` - 客户端UI弹窗

### 交互机制

1. **服务端 → 客户端**：发送 `game:salvageChoice` 事件，携带牌库顶牌信息
2. **客户端**：显示弹窗，展示卡牌，提供"超度"/"不超度"按钮
3. **客户端 → 服务端**：发送 `game:salvageResponse` 事件，携带选择结果
4. **服务端**：处理响应，更新游戏状态

### 风险点

- 需要处理玩家断线情况（超时自动选择"不超度"）
- 状态同步需确保其他玩家看到正确信息
- 牌库为空时需要特殊处理
