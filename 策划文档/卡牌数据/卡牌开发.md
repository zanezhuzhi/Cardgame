## 一、核心规则（术语）

撰写《卡牌具体文档》与实现妨害、互动类效果时，若无另行定义：

- **【对手】** 意为 **【除自己以外的其他玩家】**，**包括 AI 对手**（由 AI 代打的玩家座位与真人玩家同等视为「对手」）。
- 文案中「每名对手」「所有对手」等均以上述定义为准，不因操控者为真人或 AI 而缩小范围。
- **【弃置】**：将卡牌从牌库/手牌**直接**移入弃牌堆（**不经过已打出区**）的操作。规则文本中出现的【弃置】均按 **【主动弃置】** 判定，**不包括**回合结束清理阶段的规则弃置。完整释义见 [游戏规则说明书 §2.3](../游戏规则说明书.md#23-效果用语批注)。

### 抓牌与获得的区分

| 术语 | 定义 | 来源 | 目标位置 |
|------|------|------|----------|
| **【抓牌】** | 从**自己的牌库**抽取卡牌 | 己方牌库 | **手牌** |
| **【获得】** | 从**公共牌堆**转化为"我的牌"（牌库数量增加） | 公共牌堆（如恶评牌库） | **弃牌堆**（默认） |

**示例**：
- 「抓牌+1」：从自己牌库顶抽 1 张入**手牌**
- 「获得 1 张恶评」：从恶评牌库抽 1 张加入自己的**弃牌堆**

**注意**：
- 恶评卡、令牌等可以存在于手牌中（如通过其他效果），但**无法被使用**
- 若效果明确指定目标位置（如"加入手牌"），则按效果描述执行

### 单目标效果与可用性

- **适用范围**：卡牌或技能的效果**唯一**依赖「选择 1 个合法目标」才能完整结算，且**没有**无目标分支时，视为**纯单目标**效果。
- **规则**：当前局面下若**不存在任何合法目标**，则该卡**不可打出**，该技能**不可发动**。
- **合法目标**：以牌面/文档约定的条件为准（如生命值区间、区域等）。
- **示例（天邪鬼绿）**：场上**没有**满足条件的可退治妖怪时，**不可使用该牌**。

### 手牌上限

- **上限值**：**10 张**
- **检查时机**：每次**抓牌前**
- **规则**：若抓牌前手牌数量 **≥ 10**，则**跳过本次抓牌**

**示例（树妖效果：抓牌+2，弃置1张）**：
1. 当前 10 张手牌 → 使用树妖（剩 9 张）
2. 抓第 1 张 → 10 张
3. 抓第 2 张 → 检查 = 10，**跳过** → 仍是 10 张
4. 弃置 1 张 → 9 张

**实现说明**：
- 服务端 `drawCards` 函数在每次抽牌循环中检查 `player.hand.length >= 10`
- 若满则打印日志并跳出循环
- 后续抓牌操作不会补回被跳过的牌

## 二、公共效果规则

### 牌库展示

> 定义「查看牌库」效果的统一规则与UI表现

**基础规则**：
1. 所有玩家默认**不能查看**自己或他人的牌库内容
2. 当效果使牌库中某张牌被「展示」后，该信息将被记录
3. 展示信息为**私密信息**，仅**效果触发者**与**牌库拥有者**可见
4. 展示信息持续到该牌**离开原位置**（被抽走、弃置、超度等）

**UI表现**：

| 场景 | 牌库区域显示 |
|------|-------------|
| 无已展示牌 | 牌堆图标 + 剩余数量（不可点击） |
| 有已展示牌 | 牌堆图标 + 剩余数量（**可点击**，高亮提示） |
| 点击后弹窗 | 已展示的牌（正面） + 未展示的牌（背面 + 数量） |

**涉及卡牌**：唐纸伞妖、天邪鬼黄、赤舌、魅妖、木魅、幽谷响

## 四、文档内容规范

每张卡牌的 `.md` 文档应包含以下章节：

| 章节 | 内容 |
|------|------|
| 📋 基础信息 | 名称、HP、声誉、类型、数量、多人标记 |
| 📜 效果原文 | 摘自牌面/说明书的原始效果描述 |
| 🔍 效果分解 | 步骤拆解 + 关键规则 |
| 🎲 AI 行为决策 | 各决策点的AI策略与优先级 |
| ⚔️ 交互场景 | 典型使用场景描述 |
| ✅ 验收标准 | 基础/边界测试条件 |
| 🛠️ 实现备注 | （可选）涉及文件与注意事项 |

#### 多人标记规则

| 计算公式 | 说明 |
|----------|------|
| **标准模式（3-4人）** | 使用 `count - multiPlayer` 张 |
| **多人模式（5-6人）** | 使用全部 `count` 张 |

## 五、强制中止与AI接管

### 5.1 强制中止行为

当效果执行过程中被外部因素强制中断时的默认行为：

| 选择类型 | 默认行为 |
|----------|----------|
| 二选一 | 选择第一个选项 |
| 选择手牌 | 不选择任何牌，若必须选则选第一张 |
| 选择目标 | 选择第一个合法目标 |

**事务性原则**：
- 已生效的数值变更不回滚
- 未完成的选择步骤执行默认策略

### 5.2 AI策略等级

| 等级 | 名称 | 说明 |
|:----:|------|------|
| L0 | 随机 | 随机选择（仅用于测试） |
| L1 | 规则 | 按固定规则选择（**当前使用**） |
| L2 | 启发 | 考虑当前局面综合判断（未来扩展） |

### 7.1 问题分类速查

| 类型 | 描述 | 自查点 |
|------|------|--------|
| **逻辑不一致** | 共享层与服务端逻辑不一致 | 修改效果时**必须同步检查**两处 |
| **交互未触发** | 效果需要玩家选择，但直接执行了默认行为 | 涉及选择的效果**必须等待玩家响应** |
| **可见性遗漏** | 牌库操作后未更新展示状态 | 涉及牌库顶/底操作时**检查展示记录** |
| **响应链断裂** | 触发了交互但缺少响应处理 | 新增交互类型时**前后端必须同步** |

### 7.3 轮入道兼容检查

> **轮入道** 是游戏中最复杂的御魂，可弃置 1 张 HP≤6 的御魂并**执行其效果 2 次**。  
> 所有新开发的御魂都必须考虑被轮入道触发的场景。

#### 轮入道机制概述

| 概念 | 说明 |
|------|------|
| **队列执行** | 轮入道通过 `wheelMonkQueue` 队列管理效果执行，每次执行完整效果（包含交互选择） |
| **执行次数** | `remainingExecutions: 2`，每完成一次完整效果递减 |
| **交互等待** | 若效果触发 `pendingChoice`，队列等待响应后再继续下一次执行 |

#### 开发检查清单

**【步骤1】效果分类判断**

| 效果类型 | 示例 | 轮入道处理方式 |
|----------|------|----------------|
| **纯数值效果** | 抓牌+1、伤害+2、鬼火+1 | 直接在 `executeYokaiEffectSimple` 中处理，数值翻倍即可 |
| **自己交互效果** | 树妖（抓2选弃1）、日女巳时（三选一） | 必须在 `executeYokaiEffectByName` 中添加完整case，触发 `pendingChoice` |
| **对手交互效果** | 赤舌（对手选择）、魅妖（选对手牌） | 同上，完整执行交互流程，超时时自动AI策略 |
| **妨害无交互** | 雪幽魂（对手获得恶评） | 在 `executeYokaiEffectSimple` 中处理，效果自动对所有对手执行 |

**【步骤2】代码实现要点**

1. **有交互的御魂**：
   ```typescript
   // 在 executeYokaiEffectByName 中添加 case
   case '新御魂名': {
     // 执行效果逻辑
     // 设置 pendingChoice 等待玩家响应
     this.state.pendingChoice = {
       type: 'newYokaiChoice',  // 新增类型需同步到 server/src/types/index.ts
       playerId: player.id,
       prompt: '选择...'
     };
     break;
   }
   ```

2. **响应处理函数**：
   ```typescript
   // 在响应处理函数末尾添加队列继续调用
   public handleNewYokaiChoiceResponse(...): { success: boolean; error?: string } {
     // ... 处理响应逻辑 ...
     
     // 清除等待状态
     this.state.pendingChoice = undefined;
     
     // ⚠️ 关键：检查是否需要继续轮入道队列
     this.checkAndContinueWheelMonkQueue();
     
     this.notifyStateChange({ type: 'STATE_UPDATE', state: this.state });
     return { success: true };
   }
   ```

**【步骤3】测试验证**

- [ ] 单独使用该御魂，效果正常
- [ ] 轮入道+该御魂，效果执行2次
- [ ] 轮入道+该御魂，每次交互都正确弹出选择框
- [ ] 轮入道+该御魂，超时时自动按AI策略处理
