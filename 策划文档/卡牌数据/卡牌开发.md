# 卡牌开发流程规范

> 所有卡牌效果开发必须遵循此流程

---

## 核心规则（术语）

撰写《卡牌具体文档》与实现妨害、互动类效果时，若无另行定义：

- **【对手】** 意为 **【除自己以外的其他玩家】**，**包括 AI 对手**（由 AI 代打的玩家座位与真人玩家同等视为「对手」）。
- 文案中「每名对手」「所有对手」等均以上述定义为准，不因操控者为真人或 AI 而缩小范围。
- **【弃置】**：将卡牌从牌库/手牌**直接**移入弃牌堆（**不经过已打出区**）的操作。规则文本中出现的【弃置】均按 **【主动弃置】** 判定，**不包括**回合结束清理阶段的规则弃置。完整释义见 [游戏规则说明书 §2.3](../游戏规则说明书.md#23-效果用语批注)。

### 单目标效果与可用性（打出/发动前置）

撰写文档与实现时须遵守：

- **适用范围**：卡牌或技能的效果**唯一**依赖「选择 1 个（或 1 条）合法目标」才能完整结算，且**没有**「无目标时仍可执行的独立分支」（例如无目标则仅弃牌、无目标则改抓牌等）时，视为**纯单目标**效果。
- **规则**：在当前局面下若**不存在任何合法目标**，则该卡**不可从手牌打出**，该技能**不可发动**（UI 上应置灰或点击后明确提示「无合法目标」等；实现上须在 `canPlay`/效果入口做前置校验，**不得**先打出再弹出空选或事后 noop）。
- **合法目标**：以牌面/文档约定的条件为准（如生命值区间、区域、是否已击杀等）；仅统计**游荡妖怪区、鬼王区**等效果指定区域内**当前满足条件的对象**。
- **示例（天邪鬼绿）**：御魂效果仅为「退治 1 条」满足牌面条件的游荡妖怪时，场上**没有**任一满足条件的可退治对象时，**不可使用该牌**。

> 验收：单测/手测用例需覆盖「游荡区全空」「仅存在不满足 HP 条件」等边界，预期均为**不可打出**，且日志中不出现误触发的退治流程。

## 公共效果：牌库展示

> 定义「查看牌库」效果的统一规则与UI表现

### 基础规则

1. **默认不可查看**：所有玩家默认**不能查看自己或他人的牌库内容**
2. **展示状态记录**：当效果使牌库中某张牌被「展示」后，该信息将被记录
3. **可见性规则**：展示信息为**私密信息**，仅**效果触发者**与**牌库拥有者**可见
4. **持续时间**：展示信息持续到该牌**离开原位置**（被抽走、弃置、超度等）

### UI表现

| 场景 | 牌库区域显示 |
|------|-------------|
| 无已展示牌 | 牌堆图标 + 剩余数量（不可点击） |
| 有已展示牌 | 牌堆图标 + 剩余数量（**可点击**，高亮提示） |
| 点击后弹窗 | 已展示的牌（正面） + 未展示的牌（背面 + 数量） |

#### 牌库展示弹窗布局

```

├─────────────────────────────────────┤
│  [牌库顶]                           │

│  │ 🎴  │ ← 展示过的牌（正面显示）   │
│  │招福 │                            │
│  │达摩 │                            │
│  └─────┘                            │
│                                     │
│  [未展示] 🂠 × 11                    │
│                                     │
│  [牌库底]                           │
│  ┌─────┐                            │
│  │ 🎴  │ ← 若有置底的已知牌         │
│  └─────┘                            │
└─────────────────────────────────────┘
```

### 魅妖特殊展示

当**A玩家**使用魅妖查看**B玩家**牌库顶时：

1. **B玩家视角**：
   - 牌库区域显示：缩小的正面卡牌覆盖在牌堆上
   - 浮动提示：`🎭 [A玩家] 正在查看你的牌库顶`
   - 不抢夺交互焦点（B玩家仍可进行其他操作）

2. **A玩家视角**：
   - 弹出选择界面（已实现）
   - 选择后牌库展示状态更新

3. **其他玩家**：
   - 无法看到具体牌面（私密信息）
   - 日志显示：`A玩家 正在查看 B玩家 的牌库顶`

### 涉及卡牌

| 卡牌 | 展示触发 | 展示位置 | 展示给谁 |
|------|---------|---------|---------|
| 唐纸伞妖 | 打出时 | 自己牌库顶 | 自己 |
| 天邪鬼黄 | 打出时（置顶选择） | 自己牌库顶 | 自己 |
| 赤舌 | 对手弃牌堆→牌库顶 | 对手牌库顶 | 对手自己 |
| 魅妖 | 打出时 | 所有对手牌库顶 | 触发者+对手 |
| 木魅 | 展示直到3张阴阳术 | 自己牌库顶N张 | 自己 |
| 幽谷响 | 查看对手牌库顶 | 所有对手牌库顶 | 触发者+对手 |

### 数据结构

```typescript
interface RevealedCard {
  instanceId: string;    // 卡牌实例ID
  position: 'top' | 'bottom' | number;  // 位置（顶/底/具体索引）
  revealedBy: string;    // 谁展示的（playerId）
  revealedAt: number;    // 展示时间戳
}

// PlayerState 新增字段
revealedDeckCards: RevealedCard[];  // 该玩家牌库中已展示的牌
```

## 二、文档内容规范

每张卡牌的 `.md` 文档应包含以下章节：

### 📋 基础信息

```markdown
## 📋 基础信息

| 属性 | 值 |
|------|---|
| 名称 | xxx |
| HP | x |
| 声誉 | x |
| 类型 | 御魂/伤害/抓牌/鬼火/声誉/妨害 |
| 总数量 | N（cards.json中的count字段） |
| 多人标记 | M（cards.json中的multiPlayer字段） |
| 初始张数 | N-M（标准模式3-4人使用的数量） |
```

#### 多人标记规则

> 参考《游戏规则说明书 §4.2》：🔷纸人标记表示该卡种中有几张是5-6人多人专属

| 计算公式 | 说明 |
|----------|------|
| **标准模式（3-4人）** | 使用 `count - multiPlayer` 张 |
| **多人模式（5-6人）** | 使用全部 `count` 张 |

**示例**：

| 卡牌 | count | multiPlayer | 标准模式 | 多人模式 |
|------|:-----:|:-----------:|:--------:|:--------:|
| 赤舌 | 2 | 0 | 2张 | 2张 |
| 唐纸伞妖 | 2 | 1 | 1张 | 2张 |
| 镜姬 | 2 | 2 | 0张（不使用） | 2张 |

> ⚠️ `multiPlayer = count` 时，该卡牌**仅在多人模式使用**

### 📜 效果原文

```markdown
## 📜 效果原文

```

## 三、强制中止与AI接管

### 3.1 强制中止行为

> 定义效果执行过程中被外部因素强制中断时的默认行为

| 选择类型 | 默认行为 |
|----------|----------|
| `onChoice`（二选一） | 选择第一个选项（index=0） |
| `onSelectCards`（选择手牌） | 不选择任何牌，若必须选则选第一张 |
| `onSelectTarget`（选择目标） | 选择第一个合法目标 |

**事务性原则**：
- 已生效的数值变更不回滚
- 未完成的选择步骤执行默认策略
- 日志标注 `[超时]` 或 `[中止]`

### 3.2 AI策略等级

| 等级 | 名称 | 说明 |
|:----:|------|------|
| L0 | 随机 | 随机选择（仅用于测试） |
| L1 | 规则 | 按固定规则选择（**当前使用**） |
| L2 | 启发 | 考虑当前局面综合判断（未来扩展） |
