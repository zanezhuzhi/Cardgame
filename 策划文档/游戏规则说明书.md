## 一、游戏概述

### 1.1 游戏类型
- DBG（Deck Building Game）卡牌构筑游戏
- 3-6人对战

### 1.2 游戏模式

| 模式 | 人数 | 说明 |
|------|:----:|------|
| **标准模式** | 3-4人 | 移除带「纸人🔷」标记的卡牌 |
| **多人模式** | 5-6人 | 使用全部卡牌 |

> 达到对应人数即可开始游戏

### 1.3 胜利条件

**游戏结束触发**：
1. 当游荡妖怪牌库中翻出新的妖怪不足以补充至6张时，游戏结束
2. 当所有鬼王均被击败时，游戏立即结束

**胜利判定**：
1. 首先比较【声誉】总数，高者胜
2. 声誉相同则比较**牌库张数**
3. 再相同则判为**平局**

## 二、核心概念

### 2.1 卡牌区域

每位玩家有以下区域：

| 区域 | 说明 | 可见性 |
|------|------|--------|
| **手牌** | 当前持有的牌 | 仅自己可见 |
| **待抓取牌（牌库）** | 面朝下的牌堆 | 不可见 |
| **弃牌堆** | 打出/弃置的牌 | 公开 |
| **式神区** | 已召唤的式神 | 公开 |
| **已打出区（played）** | 本回合打出的牌，清理阶段移入弃牌堆 | 公开 |

> **手牌上限**：每位玩家手牌上限为 **10张**。当手牌达到上限时，触发抓牌效果**不会获得新的手牌**（该效果仍视为已执行，不会累积或补偿）。

> **重要概念：牌库**
> 牌库 = 待抓取牌 + 手牌 + 弃牌堆  
> 打出的牌、退治的目标均进入弃牌堆，成为牌库的一部分。  
> **超度的卡牌移出游戏，不属于牌库，不计入声誉结算。**

> **循环机制**：待抓取牌耗尽时，将弃牌堆洗牌成为新的待抓取牌（手牌不变）。

### 2.2 公共区域

| 区域 | 说明 |
|------|------|
| **游荡妖怪展示区** | 场上展示的6张妖怪，可被退治 |
| **游荡妖怪牌库** | 面朝下的妖怪牌堆 |
| **鬼王区** | 当前阶段的鬼王 |
| **恶评牌库** | 农夫+武士混洗的牌堆 |
| **阴阳术供应堆** | 基础术式/中级符咒/高级符咒 |
| **式神供应堆** | 可获取的式神 |
| **超度区** | 移出游戏的卡牌（**所有玩家公共**，任何玩家超度的卡牌都显示在这里） |

### 2.3 效果用语批注

#### 【弃置】

**定义**：将卡牌从**牌库**或**手牌**直接移入**弃牌堆**（**不经过已打出区**）的操作。

| 弃置类型 | 触发来源 | 是否触发【触】效果 | 代码标记 |
|----------|----------|:------------------:|----------|
| **主动弃置** | 牌面效果、技能、玩家选择 | ✅ 是 | `discardType: 'active'` |
| **规则弃置** | 回合结束清理阶段系统自动执行 | ❌ 否 | `discardType: 'rule'` |

**规则文本约定**：
- 规则文本（牌面效果、本规范中的效果条文等）中出现的 **【弃置】**，在条件与触发判定上 **均按【主动弃置】** 理解。
- 回合结束清理阶段的「将剩余手牌移入弃牌堆」流程属于**规则弃置**，**不触发**「当此牌被弃置时」类效果。

**示例**：
| 场景 | 弃置类型 | 三味【触】是否触发 |
|------|----------|:------------------:|
| 玩家主动选择弃置三味 | 主动弃置 | ✅ 触发抓牌+3 |
| 对手效果导致弃置三味 | 主动弃置 | ✅ 触发抓牌+3 |
| 回合结束，三味随手牌清空 | 规则弃置 | ❌ 不触发 |

> **与「打出」的区别**：打出的牌先进入「已打出区」，清理阶段再移入弃牌堆；弃置的牌**直接**进入弃牌堆，不经过已打出区。

## 三、资源系统

### 3.1 鬼火
| 属性 | 说明 |
|------|------|
| **获取** | 每回合开始时固定获得1点 |
| **使用** | 消耗鬼火以发动式神技能 |
| **上限** | **5点**（超出部分自动消失） |
| **回合结束** | **保留**（不清零） |

### 3.2 伤害
| 属性 | 说明 |
|------|------|
| **获取** | 由阴阳术、妖怪效果、式神技能产生 |
| **使用** | 在行动阶段**实时分配**给目标（不限次数） |
| **伤势** | 已分配的伤害在回合内持续生效 |
| **回合结束** | **清零**（目标生命恢复，未用伤害消失） |

### 3.3 声誉
| 属性 | 说明 |
|------|------|
| **结算时机** | 游戏结束时统计 |
| **计算范围** | 牌库（含弃牌堆）+ 式神区 |

**声誉来源**：

| 卡牌类型 | 声誉范围 | 说明 |
|----------|:--------:|------|
| 招福达摩 | +1 | 初始卡 |
| 妖怪卡 | 0 ~ +2 | 退治后进入弃牌堆 |
| 鬼王卡 | +3 ~ +5 | 击败后进入弃牌堆 |
| 式神卡 | +1 ~ +3 | 在式神区 |
| 高级符咒 | +1 | 在牌库中 |
| 恶评卡 | -1 / -2 | 农夫-1，武士-2 |
| 超度的卡 | **不计入** | 已移出游戏 |

## 五、游戏流程

### 5.1 游戏准备

#### 步骤1：准备公共牌库
1. 根据人数确定游戏模式（3-4人标准/5-6人多人）
2. 移除对应的🔷牌（标准模式移除32张妖怪+3张鬼王+6张式神）
3. 洗混游荡妖怪牌库、恶评牌库
4. 按阶段排列鬼王牌库（Ⅰ→Ⅱ→Ⅲ，麒麟在最上方）

#### 步骤2：每位玩家准备
1. 选择代表自己的**阴阳师**（玩家可自由取用阴阳师身份，作为自己参与游戏的代表。阴阳师没有特殊功能，仅用于身份代入）
2. 获得**1颗鬼火骰子**
3. 获得初始牌库：**3张招福达摩 + 6张基础术式 = 9张**
4. 洗混后抓**5张**作为起始手牌

#### 步骤3：选取式神
1. 将所有式神卡充分洗牌后，由起始玩家牌面向下随机派发
2. 每位玩家从4张式神中**选择2张**作为初始式神
3. 将选用的式神翻回正面，明示之，并列置于阴阳师卡的右侧
4. 其余2张放回式神供应堆底

> 式神上限：每位玩家最多拥有**3个**式神

#### 步骤4：场地准备
1. 翻开**5张**游荡妖怪到展示区（加上牌库顶牌共**6张**可见）
2. 翻开第一张鬼王（麒麟，无来袭效果）
3. 决定起始玩家（见下方规则）

### 5.3 玩家顺序规则

**回合顺序**：顺时针方向（从左至右），按玩家索引顺序轮转

**起始玩家**：
1. **首场游戏**：随机决定一位起始玩家
2. **连续多场游戏**：由前一场**胜方左侧**的玩家担任新的起始玩家
3. **前一场平局**：在所有**非并列第一**的玩家中随机选择起始玩家

> 游戏实现中只需维护玩家数组顺序，按索引依次执行回合即可

### 5.4 多人对局计时与托管规则（新增）

本节仅适用于**对人游戏**（多人联机，`phase=playing`）。单人模式保持现有节奏，不启用本节时控。

#### 回合倒计时（行动阶段）

- 回合倒计时显示在**游荡妖怪区上方**。
- 倒计时按当前房间内**真人玩家数量**动态确定：

| 真人玩家数 | 回合时长 |
|------------|----------|
| 1 | 无限制（与单人相同） |
| 2 | 30秒 |
| 3 | 25秒 |
| 4 | 20秒 |
| 5 | 15秒 |
| 6 | 15秒 |

- 倒计时归零时：系统自动执行该玩家的**结束回合**流程（等价于玩家点击“结束回合”）。

#### 回合外反馈与行动阶段计时暂停

- **无需等待他人时**：当前回合玩家打出牌或结算效果后，若下一步仅由**本回合玩家本人**完成选择（不依赖其他座位的操作），则**行动阶段回合计时照常进行**，连续扣除。

- **需要他人回合外反馈时**：若牌面或效果要求**其他玩家**在本回合行动阶段内做出选择（例如部分【妨害】的响应、镇墓兽由左手边玩家指定禁止退治目标等），则：
  - **当前回合玩家**界面须有明确提示（如「等待其他玩家响应…」），避免因无操作反馈而误以为卡死；
  - **行动阶段回合计时暂停**：等待他人反馈的这段时间**不计入**本回合行动限时；他人提交选择（或触发本条下述超时默认）后，若仍处于该玩家的行动阶段，**从剩余时间继续**倒计时。

- **回合外反馈限时（真人）**：由**非当前回合玩家**承担的、须在当回合内完成的上述交互，**统一限时 5 秒**（自服务端将可选项送达该玩家起算或与之等效的同步时机）。超时由系统按与 AI 托管一致的**默认策略**代为选择，对局不得无限等待。

- **AI 与离线托管座位**：若需响应的一方为 **AI** 或当前为**离线托管**，须**尽快**按既定策略或默认项完成选择（不占用真人 5 秒窗口）。

- **与「本人选择」的区分**：若交互仍指派给**当前回合玩家**（例如发动者本人点选、本回合内的目标/弃置等），则仅受**本回合行动限时**约束，**不适用**「暂停行动限时」与「5 秒回合外反馈」；以当次交互在规则与产品中实际**负责操作的座位**为准。

#### 防挂机判定

- 若玩家在对局中连续 **300秒** 无任何有效操作，判定为**退出游戏**。
- “有效操作”以服务器成功受理的对局动作为准（例如：打牌、分配伤害、使用技能、结束回合、响应选择弹窗、**在时限内完成回合外反馈提交**等）。

#### 掉线保护与离线托管

- 玩家掉线后进入“离线保护”状态，保留座位 **180秒**。
- 在 180 秒内重连成功：恢复为真人控制，继续当前对局。
- 超过 180 秒仍未回连：按**退出游戏**处理。
- 离线保护期间由 AI 托管该玩家行动；当前版本托管策略统一为 **L1**（分数区间映射后续接入）。

> 设计目的：限制单回合时长、降低对局拖延；同时在网络抖动时保护玩家可回连继续游戏。

### 5.2 回合流程

每个回合按以下阶段依次执行：

```
┌─────────────────────────────────────────────────┐
│  1. 鬼火阶段    → 鬼火+1（上限5）                 │
├─────────────────────────────────────────────────┤
│  2. 行动阶段    → 打牌、技能、式神调整、分配伤害  │
├─────────────────────────────────────────────────┤
│  3. 清理阶段    → 弃牌、补牌、补充妖怪、鬼王检查  │
└─────────────────────────────────────────────────┘
```

#### 阶段1：鬼火阶段
- 当前回合玩家获得**鬼火+1**（上限5）
- 清空上回合残余的**临时增益（TempBuff）**
- **强者离场（自动妖怪刷新）**：在当前玩家**鬼火阶段**、清空 TempBuff 之后立刻判定并结算（**无需玩家确认**）。
  - **触发条件**：紧挨着的**上一名玩家**在其刚刚结束的回合中，**未发生任意一次「击杀」**。  
    「击杀」指：将任意**游荡妖怪**或**鬼王**的生命扣至 **0**（与旧版「击败」判定一致）。仅造成伤害但未扣至 0、或只进行与击杀无关的操作，均视为未击杀。
  - **信息提示**：触发时向**房间内全体玩家**展示一条公开日志（文案见 `策划文档/界面ass/信息交互.md`）。
  - **结算步骤**：
    1. 在**游荡妖怪展示区**的 6 张游荡妖怪中，选出**当前剩余 HP 最高的 3 张**（比较的是场上该时刻的有效生命；回合开始时上一回合伤害已清空，一般为卡面生命，若有持续修改生命的效应则以场上数值为准）。
    2. **排序与平手**：将 6 张牌按**主关键字当前 HP 降序**、**次关键字展示区位置下标降序**（下标更大 = 更靠后）排序，取排序后的**前 3 张**即为离场名单。
    3. 将这 3 张按**上述排序中的先后顺序**依次置入**游荡妖怪牌库底**（后垫入的更接近牌库顶；若实现为批量移底，须与此顺序一致）。
    4. 从游荡妖怪**牌库顶**补牌，直到展示区再次满 **6** 张。
  - **不触发时**：场上游荡妖怪不变，无额外提示。
  - **设计目的**：在「上一回合无人减员」时温和刷新高强度妖怪，避免场面僵化；取消旧版「当前玩家二选一」以降低操作负担。

#### 阶段2：行动阶段
可以**任意顺序、自由组合**执行以下动作：
1. **打出手牌**：阴阳术或御魂卡（详见"手牌使用规则"）
2. **使用式神技能**：消耗鬼火发动【启】技能
3. **获得基础术式**：每回合可免费获得1张（置入弃牌堆）
4. **分配伤害**：可随时对目标分配伤害，**不限次数**（详见"伤害分配规则"）
5. **式神调整**（可选）：点击【获得式神】按钮触发，详见"式神获取与置换规则"

#### 阶段3：清理阶段
1. 将本回合已打出的牌 + 剩余手牌 → 弃牌堆
2. 【御魂/持续】类型卡可选择抓回手上
3. 从牌库抽取**5张**作为下回合手牌
4. 从游荡妖怪牌库补充妖怪至**6张**
5. 如果本回合击杀了鬼王 → 翻出下一张鬼王 → 执行【鬼王来袭】
6. 回合结束，清空伤害

### 5.5 客户端选择界面与结算提示（约定）

本节约定**电子客户端**上各类需玩家操作的弹窗（`pendingChoice` 等）的**统一布局与提示规则**，与§5.4 计时规则配合使用；不改变线下桌游规则，不影响结算结论。

#### 统一选择外壳（所有 pending 类型）

无论选择由**本回合玩家**发起，还是由**回合外玩家**响应（如部分【妨害】），弹层均应包含：

| 区域 | 内容 |
|------|------|
| **左侧** | **卡牌来源**（触发该次选择的牌：缩略图 + 名称；若无单卡来源则用约定占位，如系统/妨害标题） |
| **右上** | 默认显示**当前步骤**短说明；玩家可**展开**查看该牌**完整规则原文** |
| **右下** | **倒计时条**（见下款） |
| **下方主体区** | 具体交互（选目标、选牌、确认等），按效果类型沿用现有流程 |

#### 右下倒计时与§5.4 的对应关系

- **负责操作的人是当前回合玩家**，且该交互仅受**行动阶段回合计时**约束时：右下条与界面其他位置展示的**全局回合计时**为**同一计时**（包括仅 1 名真人时**无限制**的展示方式，与外层一致）。
- **负责操作的人是回合外玩家**，且适用§5.4 **「回合外反馈限时（真人）5 秒」**时：右下条仅表示该段**5 秒反馈**倒计时，**不与**当前回合玩家的全局行动限时混为同一条进度（避免误解）。

#### 中部淡化提示（与信息栏同源）

在一张牌**多段效果连续结算**、或从自动结算**即将进入**下一段需操作的选择时，客户端应在**信息栏公示使用的同一文案**之上，对部分玩家追加**屏幕中部短显后淡化**的提示，减少「不知道刚发生了什么」的情况。

| 情形 | 是否弹出中部提示 |
|------|------------------|
| **自己**打出的牌在本客户端进入结算提示 | **仅本人** |
| **妨害等**需**回合外**某玩家必选时 | **被点名玩家必弹** |
| 其余玩家 | **不弹中部**，默认依赖**信息栏**即可 |

> 实现与字段细节见 `docs/superpowers/specs/2026-03-29-pending-choice-ui-design.md`。

## 六、核心规则详解

### 6.1 手牌使用规则

#### 可打出的卡牌类型

| 卡牌类型 | 可否打出 | 说明 |
|----------|:--------:|------|
| 阴阳术 | ✅ | 造成伤害 |
| 御魂（妖怪/鬼王） | ✅ | 触发御魂效果 |
| 令牌 | ❌ | 招福达摩不可打出，用于超度 |
| 恶评 | ❌ | 农夫/武士不可打出 |
| 式神 | ❌ | 需要通过召唤获取 |

#### 御魂效果目标要求

**核心规则**：
- **单效果卡**：唯一效果需要合法目标才能打出，否则无法使用
- **多效果卡**：只要任一效果能执行即可打出，无法选择目标的效果自动跳过

**示例**：
| 情况 | 卡牌 | 能否打出 | 说明 |
|------|------|:--------:|------|
| 场上没有≤4HP妖怪 | 天邪鬼绿 | ❌ | 唯一效果无目标 |
| 牌库为空 | 唐纸伞妖 | ✅ | 伤害+1效果可执行 |
| 弃牌堆为空 | 座敷童子 | ✅ | 抓牌效果可执行 |
| 场上没有妖怪 | 提灯小僧 | ❌ | 唯一效果无目标 |

### 6.2 伤害分配规则（重要！）

1. **实时分配**：行动阶段内可随时分配伤害，**不限次数**
2. **伤势保留**：已分配的伤害在回合内持续生效（目标生命扣减保留）
3. 每1点伤害扣除指定目标1点生命
4. **目标选择规则**：选择目标后，持续扣除直到以下情况之一：
   - 目标生命降至0（击杀成功）
   - 剩余伤害耗尽仍不足扣除（剩余伤害归0，目标存活）
5. **可随时切换目标**：不再限制"必须击杀当前目标才能选新目标"
6. **已击杀单位处理**：
   - 生命降为0的单位**立即自动退治**，进入当前玩家的弃牌堆
   - **不再弹出退治/超度选择**（鬼王/妖怪均一律退治）
   - **回合进行中，游荡妖怪不会因离场而刷新**
8. 行动阶段结束时，所有**存活**单位生命恢复至上限，未使用伤害清空

> **线上版本优化**：相比线下"一次性分配"的限制，线上版本支持实时分配，玩家可以先打牌获得伤害、分配一部分、再打牌、再分配，更加灵活。

### 6.3 退治与超度

| 操作 | 说明 | 卡牌去向 | 声誉 |
|------|------|----------|------|
| **退治** | 将生命降为0的目标获取 | 进入自己的弃牌堆 | **计入** |
| **超度** | 将卡牌移出游戏 | 进入公共超度区 | **不计入** |

### 6.4 鬼王系统

**鬼王阶段**：Ⅰ → Ⅱ → Ⅲ

**鬼王来袭**：当新鬼王翻出时，立即执行来袭效果，影响所有玩家

**御魂效果**：鬼王卡本身也是御魂牌，打出时触发御魂效果

**特殊机制**：
- 部分鬼王具有【自】效果：回合开始时从弃牌堆回收到手牌
- 麒麟【触】：回合结束时从弃牌堆回到牌库底

### 6.5 式神技能类型

| 标记 | 名称 | 说明 |
|------|------|------|
| **【启】** | 主动技 | 消耗鬼火发动 |
| **【触】** | 触发技 | 满足条件时自动触发 |
| **【自】** | 被动触发 | 满足条件时自动执行 |
| **【永】** | 永久被动 | 只要在场就生效 |
| **【妖】** | 妖怪专属 | 作为游荡妖怪时的效果 |

### 6.6 式神获取与置换规则

> 通过在行动阶段点击【获得式神】按钮触发

#### 基础规则
- 玩家初始拥有 **2名** 式神
- 最多只能拥有 **3名** 式神
- 式神供应堆（商店）为洗混后的牌库

#### 获取式神（当前式神 < 3）

**条件**：
- 当前式神数量少于3名
- 手牌中有**至少1张「高级符咒」或「专属符咒」**
- 符咒伤害总和**≥5点**

**允许的组合**（示例）：
| 组合 | 计算 |
|------|------|
| 高级×1 + 中级×1 | 3+2=5 ✅ |
| 高级×1 + 基础×2 | 3+1+1=5 ✅ |
| 高级×2 | 3+3=6 ✅ |

> ⚠️ **必须≥5点且包含高级符咒**
> 
> ⚠️ **前期没有高级符咒时按钮不显示**，降低操作干扰
> 
> ❌ **不允许**：中级×2+基础=5（无高级）、基础×5=5（无高级）

**步骤**：
1. 选择手牌中**伤害≥5点**的阴阳术卡（必须包含至少1张高级/专属符咒）
2. 将选中的阴阳术卡置入**超度区**（移出游戏）
3. 从式神供应堆（商店）**抽取顶部2张**式神
4. 从2张中**选择1张**作为新式神
5. **展示**选中的式神，将其放置到式神区
6. 将未选中的1张放回式神供应堆**底部**

**伤害值参考**：
| 卡牌 | 伤害值 | 等级 |
|------|:------:|:----:|
| 基础术式 | 1 | 基础 |
| 中级符咒 | 2 | 中级 |
| 高级符咒 | 3 | **高级** |
| 专属符咒 | 3 | **高级** |

#### 置换式神（当前式神 = 3）

**条件**：
- 当前已拥有3名式神
- 手牌中有**1张「高级符咒」或「专属符咒」**

**唯一组合**：1张高级/专属符咒（=3点）

**步骤**：
1. 选择**1张高级/专属符咒**
2. 将该符咒置入**超度区**（移出游戏）
3. 从式神供应堆（商店）**抽取顶部2张**式神
4. 从2张中**选择1张**作为新式神
5. 选择1名**已拥有的式神**，将其放置到式神供应堆**底部**
6. **展示**选中的新式神，将其**替换**原有式神的位置
7. 将未选中的1张放回式神供应堆**底部**

> 置换只需1张高级符咒，鼓励玩家在满式神后尝试优化阵容
> 
> **流程设计说明**：先选新式神再选旧式神，让玩家在确认新式神后再决定替换哪个

#### 特殊情况

| 情况 | 处理方式 |
|------|----------|
| 式神供应堆不足2张 | 有多少抽多少（可能只有1张可选） |
| 式神供应堆为空 | 无法执行获取/置换操作 |
| 手牌符咒不足 | 无法执行（按钮置灰） |

#### 地藏像特殊获取

妖怪卡「地藏像」效果：**超度此牌，获取1个式神**
- 此效果**不消耗**额外的阴阳术卡
- 直接从式神供应堆抽取2张，选择1张
- 仍受式神上限3的限制

## 九、开发实现要点

> 以下是开发过程中确认的实现细节，供后续开发者参考

### 9.1 牌库结构

```typescript
interface PlayerState {
  deck: Card[];        // 待抓取牌
  hand: Card[];        // 手牌
  discard: Card[];     // 弃牌堆
  played: Card[];      // 本回合已打出（清理时移入discard）
  shikigami: Shikigami[];  // 式神区
  ghostFire: number;   // 鬼火（0-5）
  damage: number;      // 本回合伤害（回合结束清零）
  tempBuffs: TempBuff[];   // 临时增益（回合开始清空）
}
```

### 9.2 招福达摩特殊处理

- 招福达摩是**妖怪卡**，但**不进入游荡妖怪牌库**
- 仅用于游戏初始发给每位玩家（每人3张）
- 96张妖怪卡 = 18张招福达摩（初始） + 78张游荡妖怪

### 9.3 🔷纸人标记含义

🔷数字表示"该卡种中有几张是多人专属"，而非"整个卡种都是多人专属"

例：
- 唐纸伞妖：总数2张，🔷=1 → 1张普通+1张多人专属
- 镜姬：总数2张，🔷=2 → 2张都是多人专属（3-4人完全不用）
- 赤舌：总数2张，🔷=0 → 2张都是普通（3-4人全用）

### 9.4 恶评牌库耗尽处理

当恶评牌库剩余为0时，"获得1张恶评"效果默认获得**农夫**（无限供应）

### 9.5 效果引擎标签

```typescript
// 效果类型枚举
type EffectType = 
  | 'DRAW'           // 抓牌
  | 'DAMAGE'         // 伤害
  | 'GHOST_FIRE'     // 鬼火
  | 'DISCARD'        // 弃置
  | 'EXILE'          // 超度
  | 'CHOICE'         // 选择一项
  | 'CONDITIONAL'    // 条件效果
  | 'HINDRANCE'      // 妨害
  | 'KILL_YOKAI'     // 退治妖怪
  | 'VIEW_DECK_TOP'  // 查看牌库顶
  // ...
```

## 十一、房间聊天系统

### 11.1 功能说明

游戏内提供**实时聊天功能**，玩家可在对局过程中发送文字和表情消息，所有消息对房间内全体玩家公开可见。

### 11.2 聊天规则

| 规则 | 说明 |
|------|------|
| 可见性 | 所有聊天消息对房间内**全体玩家**公开可见 |
| 内容类型 | 纯文本 + Emoji 表情 |
| 消息长度 | 单条消息不超过 **100字符** |
| 发送频率 | 每条消息间隔至少 **5秒**（防刷屏） |
| 可用时机 | 游戏全程可用（包括等待其他玩家回合时） |

### 11.3 界面位置

聊天输入栏位于**公共信息交互区**底部（游戏日志下方、操作按钮上方），包含：
- **表情按钮**：点击弹出表情选择面板
- **文字输入框**：输入纯文本消息，支持插入表情
- **发送按钮**：发送消息（也可按 Enter 键发送）

### 11.4 消息展示

聊天消息与游戏系统日志**混排展示**在信息交互区中，以 💬 图标和玩家名前缀区分：

```
💬 [玩家名] 消息内容
```

### 11.5 GM 指令

聊天输入框同时兼作 **GM 指令入口**。以 `/` 开头的输入视为 GM 指令而非聊天消息。

| 规则 | 说明 |
|------|------|
| 触发方式 | 输入以 `/` 开头即识别为指令 |
| 可见性 | 指令及执行结果**仅对发送者可见** |
| 冷却 | GM 指令**不受** 5秒聊天冷却限制 |
| 权限 | 部分指令需要房主或开发环境权限 |

> 详细交互规范见 `策划文档/界面ass/信息交互.md` 第八章
