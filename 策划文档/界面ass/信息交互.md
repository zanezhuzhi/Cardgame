# 信息交互设计规范

> 多人模式下的游戏消息同步与展示方案

---

## 一、消息可见性规则

### 🔒 仅自己可见的消息（私有消息）

| 消息类型 | 示例 | 说明 |
|---------|------|------|
| 鬼火变化 | `🔥 回合 X - 鬼火+1（当前:Y）` | 资源敏感信息 |
| 伤害不足 | `❌ 伤害不足！` | 个人操作反馈 |
| 鬼火不足 | `❌ 鬼火不足！` | 个人操作反馈 |
| 状态错误 | `⚠️ 当前不在式神选取阶段` | 个人操作反馈 |
| 其他错误/警告 | `❌ ...` / `⚠️ ...` | 个人操作反馈 |

### 📢 同步给所有人的消息（公开消息）

除上述私有消息外，其余所有消息均同步给房间内所有玩家。

## 二、超链接引用对象

消息中的以下对象需要显示为**超链接样式**：

| 对象类型 | 示例 | 点击效果 |
|---------|------|----------|
| 卡牌（妖怪牌） | `【狰】`、`【灯笼鬼】` | 展示卡牌详情 |
| 卡牌（术式牌） | `【基础术式】`、`【中级符咒】` | 展示卡牌详情 |
| 卡牌（御魂牌） | `【破势】`、`【狂骨】` | 展示卡牌详情 |
| 式神 | `茨木童子`、`妖刀姬` | 展示式神详情 |
| 鬼王 | `酒吞童子`、`大�的�的` | 展示鬼王详情 |
| 玩家名称 | `玩家1`、`用户昵称` | 高亮显示（可选：跳转到玩家区域） |

## 三、交互行为

### 3.1 超链接样式
- 超链接文字使用**特殊颜色**（建议：`#4FC3F7` 浅蓝色 或 `#FFD700` 金色）
- 鼠标悬停时显示**下划线**，表示可点击

### 3.2 点击弹出详情
- **触发方式**：点击超链接
- **弹出位置**：在点击位置附近弹出
- **边界检测**：详情框不能超出屏幕边界，需自动调整位置
- **详情内容**：复用现有的卡牌悬浮提示框（card-tooltip）

### 3.3 关闭详情
- 点击详情框外部区域关闭
- 点击其他超链接时切换到新详情

## 四、消息数据结构

### 4.1 原有结构
```typescript
interface GameLogEntry {
  type: GameLogType;
  message: string;      // 纯文本消息
  timestamp: number;
}
```

### 4.2 新增字段
```typescript
interface GameLogEntry {
  type: GameLogType;
  message: string;      // 带占位符的消息模板
  timestamp: number;
  
  // 新增字段
  visibility: 'public' | 'private';  // 消息可见性
  refs?: {                           // 引用对象映射
    [placeholder: string]: {
      type: 'card' | 'shikigami' | 'boss' | 'player';
      id: string;           // 对象ID（卡牌instanceId、式神id等）
      name: string;         // 显示名称
      data?: any;           // 可选：完整对象数据（用于离线展示）
    }
  }
}
```

### 4.3 消息示例

**打出卡牌：**
```typescript
{
  type: 'play_card',
  message: '📜 打出【{card:狰}】，伤害+4',
  timestamp: 1711200000000,
  visibility: 'public',
  refs: {
    'card:狰': {
      type: 'card',
      id: 'yokai_zheng_001',
      name: '狰',
      data: { cardType: 'yokai', hp: 4, damage: 4, ... }
    }
  }
}
```

**击杀妖怪：**
```typescript
{
  type: 'kill',
  message: '💀 {player:玩家1} 退治了【{card:灯笼鬼}】！声誉+3',
  timestamp: 1711200001000,
  visibility: 'public',
  refs: {
    'player:玩家1': { type: 'player', id: 'p1', name: '玩家1' },
    'card:灯笼鬼': { type: 'card', id: 'yokai_lantern_002', name: '灯笼鬼', data: {...} }
  }
}
```

**私有消息（错误提示）：**
```typescript
{
  type: 'error',
  message: '❌ 鬼火不足！',
  timestamp: 1711200002000,
  visibility: 'private'
  // 无需refs
}
```

## 五、服务端消息过滤

### 5.1 广播逻辑
```
服务端广播时：
1. 遍历 state.log
2. 对于 visibility === 'private' 的消息：
   - 仅发送给 playerId 匹配的玩家
3. 对于 visibility === 'public' 的消息：
   - 发送给房间内所有玩家
```

### 5.2 实现位置
- 文件：`server/src/socket/SocketServer.ts`
- 方法：`broadcastGameState()` 需要改造，按玩家过滤日志

## 六、客户端消息渲染

### 6.1 解析超链接
```typescript
// 将 "打出【{card:狰}】" 解析为可点击的HTML
function parseLogMessage(log: GameLogEntry): VNode[] {
  const parts = log.message.split(/(\{[^}]+\})/g);
  return parts.map(part => {
    const match = part.match(/\{(\w+):([^}]+)\}/);
    if (match && log.refs?.[part.slice(1, -1)]) {
      const ref = log.refs[part.slice(1, -1)];
      return <span class="log-link" onClick={() => showDetail(ref)}>{ref.name}</span>;
    }
    return <span>{part}</span>;
  });
}
```

### 6.2 详情弹出框
- 复用现有的 `card-tooltip` 组件
- 位置计算需考虑边界检测
- 支持点击外部关闭
