# 信息交互设计规范

> 多人模式下的游戏消息同步与展示方案

---

## 一、消息可见性规则

### 🔒 仅自己可见的消息（私有消息）

| 消息类型 | 示例 | 说明 |
|---------|------|------|
| 鬼火变化 | `🔥 回合 X - 鬼火+1（当前:Y）` | 资源敏感信息 |
| 伤害不足 | `❌ 伤害不足！` | 个人操作反馈 |
| 鬼火不足 | `❌ 鬼火不足！` | 个人操作反馈 |
| 状态错误 | `⚠️ 当前不在式神选取阶段` | 个人操作反馈 |
| 其他错误/警告 | `❌ ...` / `⚠️ ...` | 个人操作反馈 |

### 📢 同步给所有人的消息（公开消息）

除上述私有消息外，其余所有消息均同步给房间内所有玩家。

## 二、超链接引用对象

消息中的以下对象需要显示为**超链接样式**：

| 对象类型 | 示例 | 点击效果 |
|---------|------|----------|
| 卡牌（妖怪牌） | `【狰】`、`【灯笼鬼】` | 展示卡牌详情 |
| 卡牌（术式牌） | `【基础术式】`、`【中级符咒】` | 展示卡牌详情 |
| 卡牌（御魂牌） | `【破势】`、`【狂骨】` | 展示卡牌详情 |
| 式神 | `茨木童子`、`妖刀姬` | 展示式神详情 |
| 鬼王 | `酒吞童子`、`大�的�的` | 展示鬼王详情 |
| 玩家名称 | `玩家1`、`用户昵称` | 高亮显示（可选：跳转到玩家区域） |

## 三、交互行为

### 3.1 超链接样式
- 超链接文字使用**特殊颜色**（建议：`#4FC3F7` 浅蓝色 或 `#FFD700` 金色）
- 鼠标悬停时显示**下划线**，表示可点击

### 3.2 点击弹出详情
- **触发方式**：点击超链接
- **弹出位置**：在点击位置附近弹出
- **边界检测**：详情框不能超出屏幕边界，需自动调整位置
- **详情内容**：复用现有的卡牌悬浮提示框（card-tooltip）

### 3.3 关闭详情
- 点击详情框外部区域关闭
- 点击其他超链接时切换到新详情

### 3.4 信息区滚动交互

#### 滚动条样式

| 属性 | 值 | 说明 |
|------|:--:|------|
| 滚动条宽度 | **6px** | 纤细滚动条，不占用过多空间 |
| 滚动条颜色 | `#D4A574` | 与游戏主题金色一致 |
| 滚动条背景 | `transparent` | 透明背景，仅显示滑块 |
| 滚动条圆角 | `3px` | 圆润外观 |
| 悬浮时宽度 | **8px** | 悬浮时略微变粗，增强可操作性 |

#### 滚动触发方式

| 触发方式 | 说明 |
|----------|------|
| 鼠标滚轮 | 在信息区任意位置滚动 |
| 拖拽滚动条 | 拖拽滑块上下滚动 |
| **悬浮滚动区域** | 鼠标悬浮在滚动条区域（右侧20px）时，滚轮可触发滚动 |

#### 消息存储限制

| 限制项 | 值 | 说明 |
|--------|:--:|------|
| 最大消息回溯数 | **300条** | 超过300条时自动删除最早的消息 |
| 新消息自动滚动 | 是 | 有新消息时自动滚动到底部（除非用户正在查看历史） |
| 历史消息持久化 | 否 | 房间销毁后消息不保留 |

#### 滚动行为细节

| 行为 | 说明 |
|------|------|
| 用户查看历史时 | 新消息到达**不自动滚动**，避免干扰阅读 |
| "新消息"提示 | 有未读新消息且用户在查看历史时，底部显示"↓ 新消息"按钮 |
| 点击"新消息" | 滚动到最新消息位置 |

## 四、消息数据结构

### 4.1 原有结构
```typescript
interface GameLogEntry {
  type: GameLogType;
  message: string;      // 纯文本消息
  timestamp: number;
}
```

### 4.2 新增字段
```typescript
interface GameLogEntry {
  type: GameLogType;
  message: string;      // 带占位符的消息模板
  timestamp: number;
  
  // 新增字段
  visibility: 'public' | 'private';  // 消息可见性
  refs?: {                           // 引用对象映射
    [placeholder: string]: {
      type: 'card' | 'shikigami' | 'boss' | 'player';
      id: string;           // 对象ID（卡牌instanceId、式神id等）
      name: string;         // 显示名称
      data?: any;           // 可选：完整对象数据（用于离线展示）
    }
  }
}
```

### 4.3 消息示例

**打出卡牌：**
```typescript
{
  type: 'play_card',
  message: '📜 打出【{card:狰}】，伤害+4',
  timestamp: 1711200000000,
  visibility: 'public',
  refs: {
    'card:狰': {
      type: 'card',
      id: 'yokai_zheng_001',
      name: '狰',
      data: { cardType: 'yokai', hp: 4, damage: 4, ... }
    }
  }
}
```

**击杀妖怪：**
```typescript
{
  type: 'kill',
  message: '💀 {player:玩家1} 退治了【{card:灯笼鬼}】！声誉+3',
  timestamp: 1711200001000,
  visibility: 'public',
  refs: {
    'player:玩家1': { type: 'player', id: 'p1', name: '玩家1' },
    'card:灯笼鬼': { type: 'card', id: 'yokai_lantern_002', name: '灯笼鬼', data: {...} }
  }
}
```

**私有消息（错误提示）：**
```typescript
{
  type: 'error',
  message: '❌ 鬼火不足！',
  timestamp: 1711200002000,
  visibility: 'private'
  // 无需refs
}
```


### 6.1 解析超链接
```typescript
// 将 "打出【{card:狰}】" 解析为可点击的HTML
function parseLogMessage(log: GameLogEntry): VNode[] {
  const parts = log.message.split(/(\{[^}]+\})/g);
  return parts.map(part => {
    const match = part.match(/\{(\w+):([^}]+)\}/);
    if (match && log.refs?.[part.slice(1, -1)]) {
      const ref = log.refs[part.slice(1, -1)];
      return <span class="log-link" onClick={() => showDetail(ref)}>{ref.name}</span>;
    }
    return <span>{part}</span>;
  });
}
```

## 八、房间聊天系统

> 玩家在游戏内进行实时文字+表情交流，消息对房间内所有玩家公开可见

### 8.1 功能概述

在公共信息交互区底部（游戏日志与操作按钮之间）增加**聊天输入栏**，允许玩家发送纯文本和表情包消息。聊天消息与游戏日志混排显示于信息交互区中，但通过样式区分。

| 限制项 | 值 | 说明 |

#### 表情选择流程

1. 点击输入栏左侧 **😀 表情按钮**

### 8.7 通信协议

#### 客户端 → 服务端

```typescript
// Socket事件
socket.emit('chat:send', {
  content: string,       // 消息内容
  roomId: string         // 房间ID
});
```

#### 服务端处理

1. 校验消息合法性（长度、频率、内容过滤）
2. 构造 `GameLogEntry`（type: 'chat'）
3. 写入 `state.log`
4. 广播给房间内所有玩家（visibility: 'public'）

#### 服务端 → 客户端

聊天消息作为 `GameLogEntry` 的一部分，随 `state.log` 同步广播，无需额外事件通道。

### 8.9 GM 指令系统

聊天输入框同时作为 **GM 指令**的入口。以 `/` 开头的输入视为 GM 指令，不作为聊天消息发送。

#### 指令格式

```
/指令名 [参数1] [参数2] ...
```

#### 判定规则

| 条件 | 处理方式 |
|------|----------|
| 输入以 `/` 开头 | 视为 GM 指令，进入指令处理流程 |
| 输入不以 `/` 开头 | 视为普通聊天消息，走聊天发送流程 |

#### 权限控制

| 指令类型 | 权限要求 | 说明 |
|----------|----------|------|
| 调试指令 | 仅开发/测试环境可用 | 如 `/give`、`/setfire` 等 |
| 管理指令 | 房主权限 | 如 `/kick`、`/pause` 等 |
| 信息指令 | 所有玩家可用 | 如 `/help`、`/status` 等 |

#### 指令响应

- GM 指令**不广播**给其他玩家，仅对发送者显示执行结果
- 指令结果以系统消息格式显示在日志区：`⚙️ [GM] 执行结果...`
- 无效指令返回提示：`⚙️ [GM] 未知指令，输入 /help 查看可用指令`

#### 通信协议

```typescript
// 客户端判定
function handleSend(input: string) {
  if (input.startsWith('/')) {
    socket.emit('gm:command', {
      command: input.slice(1),   // 去掉 '/' 前缀
      roomId: string
    });
  } else {
    socket.emit('chat:send', {
      content: input,
      roomId: string
    });
  }
}
```

```typescript
// 服务端处理
socket.on('gm:command', ({ command, roomId }) => {
  const [cmd, ...args] = command.split(' ');
  const result = executeGMCommand(cmd, args, player, room);
  // 仅回复给发送者
  socket.emit('gm:result', {
    message: result.message,
    success: result.success
  });
});
```

#### 交互细节
