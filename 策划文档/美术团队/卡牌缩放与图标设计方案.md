### 1.4 缩放限制

#### 最小/最大缩放限制

| 资源类型 | 最小缩放 | 最大缩放 | 理由 |
|---------|---------|---------|------|
| 竖向卡牌 | 50% | 150% | 保持可读性和视觉清晰 |
| 鬼王卡 | 80% | 140% | 特殊卡牌，缩放范围较小 |
| 式神卡 | 50% | 100% | 基础尺寸已缩小（75%），不宜再小 |
| 头像 | 80% | 150% | 保证面部特征可识别 |

#### 缩放原则

1. **等比例缩放**: 所有缩放必须保持原始宽高比，不拉伸变形
2. **整数像素**: 避免半像素导致模糊，缩放后尺寸取整
3. **对齐网格**: 缩放后需对齐到UI网格，避免错位
4. **性能优先**: 实时缩放使用CSS transform，避免频繁重绘

### 1.5 技术实现建议

#### CSS Transform 方式（推荐）

```css
/* 基础类 */
.card-base {
  width: 200px;
  height: 280px;
  transition: transform 0.2s ease; /* 已调整：0.2s */
}

/* 缩放变体 */
.card-shiki {
  width: 150px;  /* 已调整：75%缩放 */
  height: 210px;
}

.card-boss {
  width: 540px;
  height: 380px;
}

/* 悬停效果 */
.card-base:hover {
  transform: scale(1.1);
  z-index: 10;
}

/* 手牌区悬停特殊处理 */
.hand-card:hover {
  transform: scale(1.1);
  z-index: 15; /* 手牌区悬停更高层级 */
}

/* 头像悬停 */
.avatar:hover {
  transform: scale(1.1);
  box-shadow: 0 0 15px rgba(212, 165, 116, 0.8);
}
```

#### Background-Size 方式（卡背/图标）

```css
/* 卡背统一 */
.card-back {
  background-image: url('card_back.png');
  background-size: contain;
  background-repeat: no-repeat;
  background-position: center;
}

/* 卡背缩放变体 */
.card-back--shiki {
  width: 150px;  /* 已调整：75%缩放 */
  height: 210px;
}

.card-back--boss {
  width: 540px;
  height: 380px;
}
```

#### JavaScript 动态缩放（弹窗预览）

```javascript
// 弹窗放大预览
function showCardPreview(card) {
  const preview = document.getElementById('card-preview');
  const scale = 1.4; // 140%放大
  
  preview.style.width = `${card.width * scale}px`;
  preview.style.height = `${card.height * scale}px`;
  preview.style.backgroundImage = `url(${card.image})`;
}
```

#### 性能优化建议

1. **使用GPU加速**: `transform: translateZ(0)` 强制GPU渲染
2. **will-change**: 提前告知浏览器动画属性
3. **requestAnimationFrame**: 动画使用RAF而非setInterval
4. **懒加载**: 弹窗预览时才加载大图

## 方案二：缺失图标设计方案

| 图标名称 | 推荐尺寸 | 当前状态 | 优先级 |
|---------|---------|---------|:------:|
| **伤害值图标** | 40×40 | 占位灰色 #D9D9D9 | ⭐⭐⭐ |

### 2.2 设计规范

#### 尺寸标准

**统一尺寸：40×40**

- 理由：
  - 与现有图标（鬼火、牌数、声望）保持一致
  - 视觉平衡：所有图标大小相同
  - 对齐方便：图标数值框尺寸为 85×60（包含40×40图标+30×32数字）

**特殊尺寸（可选）：**
- 头像框：120×120（已有）
- 按钮：150×120（已有）

#### 视觉风格

**日式和风风格：**

1. **线条风格**：
   - 使用单线或双线勾勒轮廓
   - 线条粗细：2px（与UI边框一致）
   - 伤害值图标：**2.5px**（已调整，提高可见性）
   - 转角圆润：半径 4-6px

2. **配色方案**：
   - 主色：金色 `#D4A574`（与边框一致）
   - 辅助色：深红 `#782B2B`（警示）
   - 背景：透明或深色半透明

3. **装饰元素**：
   - 日式纹样：云纹、麻叶、樱花
   - 神道教符号：勾玉、御守、封印
   - 和风边框：双线、波浪线

### 2.4 与现有图标的风格统一性

### 2.6 图标交付规范

#### 文件格式

| 资源类型 | 格式 | 理由 |
|---------|------|------|
| **矢量图标** | .svg | 可缩放、文件小、支持动画 |
| **位图图标** | .png | 兼容性好、透明支持 |
| **WebP** | .webp | 高压缩率、支持透明（可选） |

#### 导出倍率

- **@1x**: 40×40（实际使用）
- **@2x**: 80×80（高清屏）
- **@3x**: 120×120（超高清屏，可选）

#### 命名规范

```
图标命名：icon_[用途].svg
示例：
- icon_damage.svg
- icon_deck.svg
- icon_discard.svg

变体命名：icon_[用途]_[状态].svg
示例：
- icon_damage_hover.svg
- icon_deck_disabled.svg
```

#### 图层组织（Figma/Sketch）

```
├── Background (optional)
├── Main Shape
│   ├── Stroke (2px, #D4A574)
│   └── Fill (transparent or solid)
├── Decoration
│   └── Pattern (云纹/樱花等)
└── Center Element
    └── Icon (刀剑/阴阳/封印等)
```
