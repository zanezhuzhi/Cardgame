# 御魂传说 - 美术团队工作流程

> 版本：v1.0.0  
> 创建日期：2026-03-21

---

## 工作流总览

```
制作人提供设计方案
        │
        ▼
【阶段1】交互UI设计师
  完善界面交互细节
  输出：交互需求文档
        │
        ▼
  制作人审核 ──────→ 修改 ──────→ 返回交互UI设计师
        │（通过）
        ▼
【阶段2】GUI设计师
  评估交互细节 + 输出视觉设计效果
        │
        ▼
  ┌─────────────────────┐
  │  美术总监  ×  制作人助手  │  ← 交叉审核
  └─────────────────────┘
        │（双方意见汇总）
        ▼
  制作人最终确认 ──→ 修改 ──→ 返回GUI设计师
        │（通过）
        ▼
【阶段3】资产管理师
  拆解美术资源需求列表
  明确资源规格及使用细节
  发送需求给：美术设计师 + 动效设计师
        │
        ▼
  ┌─────────────────────┐
  │  美术总监  ×  制作人助手  │  ← 交叉审核
  └─────────────────────┘
        │（双方意见汇总）
        ▼
  制作人最终确认 ──→ 修改 ──→ 返回资产管理师
        │（通过）
        ▼
【阶段4】美术设计师 + 动效设计师（并行）
  美术设计师：产出静态美术资源
  动效设计师：产出动效资源规范
        │（同步产出）
        ▼
【阶段5】资产管理师
  规格验收所有资源
  输出验收报告
        │
        ▼
  输出给制作人 ✅
```

---

## 各阶段详细说明

### 阶段1：交互需求设计

| 项目 | 内容 |
|------|------|
| 负责人 | 交互UI设计师 |
| 输入 | 制作人设计方案 |
| 输出 | 交互需求文档（流程图 + 状态说明 + 边界情况） |
| 审核 | 制作人直接审核 |
| 通过条件 | 制作人明确表示"通过"或"进入下一阶段" |

---

### 阶段2：视觉设计

| 项目 | 内容 |
|------|------|
| 负责人 | GUI设计师 |
| 输入 | 已通过的交互需求文档 |
| 输出 | 视觉设计规范 + 界面效果描述 |
| 审核 | 美术总监 + 制作人助手（交叉审核）→ 制作人最终确认 |
| 通过条件 | 两方审核意见无重大异议，制作人确认 |

---

### 阶段3：资产需求拆解

| 项目 | 内容 |
|------|------|
| 负责人 | 资产管理师 |
| 输入 | 已通过的视觉设计规范 |
| 输出 | 资源需求清单（含规格、数量、用途、命名规范） |
| 审核 | 美术总监 + 制作人助手（交叉审核）→ 制作人最终确认 |
| 通过条件 | 资源无遗漏，规格合理，制作人确认 |

---

### 阶段4：资源产出（并行）

| 项目 | 内容 |
|------|------|
| 负责人 | 美术设计师（静态）+ 动效设计师（动态）|
| 输入 | 已通过的资源需求清单 |
| 输出 | 美术资源 + 动效规范文档 |
| 审核 | 资产管理师规格验收 |
| 通过条件 | 资源规格符合需求清单 |

---

### 阶段5：资源验收输出

| 项目 | 内容 |
|------|------|
| 负责人 | 资产管理师 |
| 输入 | 美术设计师 + 动效设计师的产出 |
| 输出 | 最终验收报告 + 资源交付清单 |
| 接收方 | 制作人 |

---

## 沟通规范

- 每个 Agent 完成任务后，明确标注"**提交审核**"并说明提交给谁
- 审核方收到后，明确给出"**通过**"或"**修改意见：[具体内容]**"
- 制作人的每次确认或决策，由制作人助手记录到 `制作人语录.md`
- 所有文档存放路径：`d:\AI桌游卡牌\Cardgame\Cardgame\策划文档\美术团队\`
