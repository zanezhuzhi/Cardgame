# 御魂传说 - 项目思维导图

> **最后更新**: 2026-03-26
> **项目类型**: 《阴阳师》IP衍生线上卡牌游戏（DBG，3-6人对战）
> **技术栈**: Vue3 + Node.js + Socket.io + TypeScript

---

## 🎮 一、项目概览

```
御魂传说（百鬼夜行）
├── 📁 项目定位
│   ├── 类型：DBG卡牌构筑游戏
│   ├── 玩家人数：3-6人对战
│   ├── 游戏模式：标准模式（3-4人）+ 多人模式（5-6人）
│   └── 胜利条件：声誉最高 → 牌库最多 → 平局
│
├── 📁 核心机制
│   ├── 回合流程：鬼火阶段 → 行动阶段 → 清理阶段
│   ├── 牌库系统：待抓取牌 + 手牌 + 弃牌堆（循环机制）
│   ├── 鬼火资源：行动点数（初始1点，上限10点）
│   ├── 声誉系统：获取式神+1，退治妖怪+声誉值
│   └── 超度机制：卡牌移出游戏，不计入牌库
│
└── 📁 美术风格
    ├── 主色调：深紫蓝背景 + 金色边框
    ├── 风格：日式和风 + 二次元唯美
    ├── 参考：《阴阳师》手游调性
    └── 关键词：妖怪、唯美、和风、二次元
```

## 🎴 三、卡牌体系

```
卡牌总数：400张
│
├── 🟣 鬼王卡 (10张)
│   ├── 编号：boss_001 ~ boss_010
│   ├── 触发条件：鬼火阶段，打出指定御魂时触发
│   └── 效果：arrival（来袭）效果 + 退治奖励
│
├── 🟡 妖怪卡 (32类，78张)
│   ├── 编号：yokai_001 ~ yokai_032
│   ├── 游荡妖怪：38种不同妖怪
│   ├── 招福达摩：18张令牌（获取后直接弃置获得奖励）
│   └── 效果：御魂效果（打出时触发）+ 退治奖励
│
├── 🔵 式神卡 (24张)
│   ├── 编号：shikigami_001 ~ shikigami_024
│   ├── 多人专属：6张（标准模式不使用）
│   ├── 获取条件：5声誉 + 含高级符咒
│   └── 效果：被动技能（每回合自动触发）
│
├── 🟢 阴阳术卡 (80张)
│   ├── 编号：spell_001 ~ spell_003
│   ├── 三种级别：基础术式（1鬼火）→ 中级符咒（2鬼火）→ 高级符咒（3鬼火）
│   ├── 效果：提供鬼火、声誉、伤害等资源
│   └── 超度效果：可超度弃牌堆的妖怪来升级符咒
│
├── ⚫ 恶评卡 (30张)
│   ├── 编号：penalty_001, penalty_002
│   ├── 类型：农夫（声誉-1）、武士（声誉-2）
│   └── 触发：退治妖怪时随机抽取
│
└── 🟤 阴阳师 (6张)
    ├── 编号：onmyoji_001 ~ onmyoji_006
    ├── 初始牌：每位玩家2张
    └── 效果：基础能力（提供鬼火、声誉、伤害）
```

## 🔧 四、开发工作流

```
📋 六步TDD开发流程
│
├── 1️⃣ 文档确认
│   └── 更新策划/技术文档 → 需求方确认范围与验收标准
│
├── 2️⃣ 编写代码
│   └── 实现功能逻辑（类型定义、核心方法、UI渲染）
│
├── 3️⃣ 建立TDD用例
│   └── 编写测试用例 → 先失败(Red)后通过(Green)
│
├── 4️⃣ 自动化测试
│   └── cd shared && npm test → 直至全部通过
│
├── 5️⃣ 创建GM指令
│   └── 添加调试指令协助手工测试
│
└── 6️⃣ 提交并重启
    └── 重启服务端/客户端 → 人工跟进测试
```
